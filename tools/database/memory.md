# Database Agent — Memory

## 1. Schema版本历史

### 版本变更日志

#### v1.3.0 (2024-01-14)
- **变更**: 添加 `knowledge_entries` 表的 `confidence` 字段
- **原因**: 支持知识可信度评分
- **迁移**: `ALTER TABLE ADD COLUMN confidence DECIMAL(3,2) DEFAULT 1.00`
- **回退**: 可安全回退，不影响现有数据

#### v1.2.0 (2024-01-10)
- **变更**: `deliverables` 表添加 `file_hash` 字段
- **原因**: 支持文件完整性校验
- **迁移**: 添加列 + 存量数据批量计算SHA256
- **回退**: 不影响现有查询

#### v1.1.0 (2024-01-08)
- **变更**: `tasks` 表添加 `estimated_hours` 和 `actual_hours` 字段
- **原因**: 支持工时追踪
- **迁移**: 添加列，默认NULL
- **回退**: 可安全回退

#### v1.0.0 (2024-01-05)
- **初始版本**: 核心五表创建
- **表**: projects, tasks, agents, deliverables, reviews
- **索引**: 基础查询索引

### Schema版本管理表

```sql
CREATE TABLE schema_versions (
    version     VARCHAR(10) PRIMARY KEY,
    applied_at  TIMESTAMPTZ DEFAULT NOW(),
    description TEXT,
    migration_script TEXT,
    rollback_script TEXT,
    applied_by  VARCHAR(64)
);
```

### 优化记录

#### 2024-01-12: 项目时间线查询优化
- **问题**: 项目完整时间线查询450ms
- **方案**: 添加复合索引(project_id, created_at) + 覆盖索引
- **效果**: 优化至80ms，提升82%

#### 2024-01-10: Agent活动聚合优化
- **问题**: Agent活动查询涉及多表JOIN，320ms
- **方案**: 创建物化视图，每5分钟刷新
- **效果**: 优化至45ms，提升86%

#### 2024-01-08: 知识库搜索优化
- **问题**: 全文检索180ms
- **方案**: tsvector预计算 + GIN索引
- **效果**: 优化至25ms，提升86%
