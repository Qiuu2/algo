# 静态缓冲差分指向性测试(软件法)— 分开 🅰 固件撕裂 vs 🅱 板 DAC 极性

> **目的**:无示波器、一步分开 **🅰(TX DMA 读写撕裂=固件)** 与 **🅱(板 DAC 模拟输出翻极性=硬件)**。
> **原理**:把 TX 缓冲写成**静态固定波形、让 beam 不再覆盖它** → 🅰(写竞争)**在构造上不可能发生** → 测指向性:**静态变指向=🅰;静态仍平=🅱**。
> **性质**:诊断用固件改动(宏门控、可逆、不碰冻结 FIRA/算法)+ 台前声学测量。**CTO-gated**(改代码+驱动真喇叭)。
> **来源**:board-output-defect-hunt workflow + PM,**独立 critic(claude-opus-4-8)过门:机制/波形数学判对,修 8 MAJOR→本 v1**。波形 python 双轨核准。

---

## 0. 判据预告(先知道要看什么)
| 结果 | 结论 | 下一步 |
|---|---|---|
| 静态**指向**(±90°落差≥~10dB,像竞品同1m) + 活波束**平** | **🅰 固件撕裂**(去掉写时序就好) | 修固件缓冲/半区时序(几行) |
| 静态**也平**(和活一样) | **🅱 板 DAC 极性/走线** | 换 375Hz 静态表逐路定位翻相通道→改板子接线 |

## 1. build 组合与上板前提(critic CAVEAT-A / punch-3)
- **必须 `M2_FIRA_INLOOP=1` 且 `M2_STATIC_TXTEST=1` 一起**。⚠ M1 build(`=0`)的 RX 回调每帧把 mono 扇出重写全部 8 路 TX → **不是静态,不能用**。
- **上板 gate(FIRA setup 必须成功,否则 SPORT 没 arm=无声,测试无意义)**:`g_m1_valid==1` 且 `g_m2_setup_rc==0` 且 `g_m1_rx_block_count` 在增长。不满足→停,先修 build。
- 阵列**横放**、broadside 正前(已确认)。

## 2. 代码改动(诊断,零板上运算,宏一关字节还原)
1. **表**:`#include "m2_static_txtest_table.h"`(python 预生成的最终 Q31 值,**板上不重算**,同源纪律)。
2. **填充**(两半相同,64 样本=3 周期自身无缝):
   ```c
   #if M2_STATIC_TXTEST
   memcpy(s_m1_tx_buf[0], M2_STXT_TBL, sizeof(M2_STXT_TBL));  /* 512 int32 = 一个半区 */
   memcpy(s_m1_tx_buf[1], M2_STXT_TBL, sizeof(M2_STXT_TBL));  /* 两半相同 */
   #endif
   ```
   插入点:**最简**=`m1_loopback_init()` 返回后、`while(1)` 前调用一次(有 <3ms 一次性启动瞬态:静音→音,无害);**更干净**=放进 init 里 `adi_sport_Enable` 之前(无瞬态)。
3. **跳过 beam**(M2 下唯一写 TX 的地方;RX 回调 keep-last 不碰 TX):
   ```c
   #if !M2_STATIC_TXTEST
       m2_beam_poll();
   #endif
   ```
4. **字节一致纪律(critic MAJOR)**:宏 `M2_STATIC_TXTEST` 未定义时,预处理输出与现 M2/M1 build **token 相同**(照 R57 `M2_RX_RIGHT_ALIGNED` 先例);文件 0 非 ASCII。
> 静态模式下 RX 回调仍跑、`g_m2_overrun_count` 会一直涨(poll 没消费)——**这是正常的,别当故障**。

## 3. 波形(已核,不用改)
- 2250Hz(=3 周期/64 帧,自身无缝;<3118Hz 无空间混叠;L=5.4λ 够指向)。−12.04dBFS,不削波。同相 Dolph 加权,与真波束同格式(c7=输入低字节00,其余=×Dolph)。
- **为什么活波束也用 2250Hz、且和静态等价**:FIRA 滤波对 8 路是**共模**,不改通道间相对权重/相位 → 过 FIRA(活)与绕过(静)**指向性等价**。(这句要留着,堵"你没测真波束"的质疑。)

## 4. 步骤(含 critic 的 4 道防呆门)
**① 活波束基线(HOLE-3 硬门)**:正常 M2、喂 2250Hz 进 ADC,测 0°/±30/±60/±90°(水平,每角 3 点中值)。**必须复现"平 ~3dB"**;**若 2250Hz 下不平 → 停**(缺陷可能只在 2kHz,差分会被混淆)。

**② 编 `M2_FIRA_INLOOP=1 && M2_STATIC_TXTEST=1`,烧板。**

**③ 存活 + 格式门(HOLE-1 / HOLE-4,防"假平=假🅱")**:
- (a) **听**:静态音要出声;
- (b) **gate**:`g_m1_valid=1` / `g_m2_setup_rc=0` / `g_m1_rx_block_count` 涨 / `g_m1_fg_stream_live=1`。**⚠ `g_m2_fg_beam_live=-99`、`g_m2_out_nonzero=0` 是静态模式的正常"诚实死"(poll 没跑),不是失败,别用波束 FG 门判它**;
- (c) **JTAG 回读 `s_m1_tx_buf` 帧 16**,8 值必须对上下表(**全同号 + Dolph 比值 + c7 低字节 00**)。任一不符=fill/格式错 → 停,别测(否则"静态平"是填错造成的假 🅱):

  | c | 期望值(帧16,hex) | 比值/c7 | 符号 |
  |---|---|---|---|
  | c0 | 0xe4430000 | 0.8668(抬边) | 负 |
  | c1 | 0xefdcc000 | 0.5043(最小) | 负 |
  | c2 | 0xec1b4000 | 0.6217 | 负 |
  | c3 | 0xe8884000 | 0.7334 | 负 |
  | c4 | 0xe55a4000 | 0.8327 | 负 |
  | c5 | 0xe2c48000 | 0.9135 | 负 |
  | c6 | 0xe0f18000 | 0.9705 | 负 |
  | c7 | 0xe0000000 | 1.0000(最大) | 负,低字节00 |
  (帧16 是正弦谷点→8 值全负、这正常;关键是**同号 + 比值=Dolph + c7 低字节00**。)

**④ 信噪比门(HOLE-2,防"太安静=假平")**:先测本底噪声;功放**最小起、逐步升到 0° SPL 高出本底 ≥~15dB**(太安静→±90° 触底→看着平=假 🅱)。安全:逐步升,刺耳/破音即停;连续纯音留意喇叭发热。

**⑤ 测静态波束**:0°/±30/±60/±90°(水平,每角 3 点中值,近场散斑→要**可重复**,不认单点)。**静态测试不需要 ADC 输入**(TX 与 RX 独立)。

**⑥ 判定**(见 §0 表):
- 静态**指向**(±90° 落差 ≥~10dB;此 10dB 是**经验值、锚定竞品同 1m 的 12-16dB**,不是远场规格)+ 活**平** → **🅰**;
- 静态**也平** → **🅱**。
- 差分为什么在 1m 近场成立:近场效应对"活/静态"两次是**共模**(同房间/麦/距离/角度/音量),相减抵消——所以 §2 那句"完全相同的设置"是承重的,别省。

**⑦ 若判 🅱 → 定位(critic punch-8:换 375Hz,别用 2250!)**:2250Hz 在 1m 近场会把外侧对误判成翻相。判 🅱 后我另给一版 **375Hz 静态表**(128 帧 1 周期,EXP_PLAN 300-400Hz 口径)+ 逐通道 mute,用 EXP_PLAN 的 C0/C2 逐路比 c7 找出翻相通道。

**⑧ 关宏,字节还原,恢复正常 build。**

## 5. 安全 / 合规
- 功放最小起、逐步升;连续 2250Hz 纯音**限电平+限时长**(比节目信号更烤喇叭,而且为信噪比会调高),CTO 操作清单里写明,留意发热/失真。
- 宏默认 OFF;测完还原;**不碰 `.cproject`/`.project`**(防测试员 git 冲突);**不把诊断 build 提交主线**;冻结 FIRA/算法零触碰(fill 写的是非冻结 TU 里的 `s_m1_tx_buf`)。
- fill 下标已核:`63*8+7=511<512`,无越界。
