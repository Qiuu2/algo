## ⚠ 0. 先读:这是**假设**,远场 A/B 是唯一坐实门

- **锚点强度不均**:Phase A(375 solo)里**只有 C0=最外(边)、C7=正中(中心)是 user 亲耳定位的实锚 [L1]**;中间 C1–C6 的排序是**假设单调 [L2]**。所以修法方向(把最大权重搬回中心、边缘搬回边)是硬的,中间几对的精确顺序不是。
- **近场会把差异压扁**:今天 2k@30° 只差 2dB 就是 1m 近场的假象;**远场(见 §3)才看得到预测的 7~13dB**。**1m 测出来无论好坏都不作结论**——只能作"最长可用距离的相对趋势"。
- **rig 特定**:C2/C4 极性 + 本映射都是"**竞品喇叭 + 我们接线**"这套 rig 的问题;换回自家阵列要重验(装配 QA)。
- 因此:**本测 PASS 的判据 = 远场 A/B 里 B 臂 30° 明显更抑制**(§6),不是"更窄"(反而略宽,见下)。

## 1. build 前提
- **必须带 `-DM2_CHMAP_FIX`**(build GUI 里 `-D` 加,**不进 `.cproject`**);先 clean build 过编译再烧。
- **不加这个宏 = 预处理输出与今天固件逐 token 相同**(默认 `#else` 走 `g_dolph_w8_q15[c]`)→ **测了白测**,务必确认加了。
- 其余上板 gate 同 M2 常规:`g_m1_valid==1` / `g_m2_setup_rc==0` / `g_m1_rx_block_count` 增长 / `g_m1_fg_stream_live==1` / `g_m2_fg_beam_live==1`(M2 活波束模式,poll 在跑)。
- 阵列**横放**、broadside 正前。喂**单 mono** 信号。

## 2. 核心方法 —— 单 build,JTAG 活切 A/B(最干净,和钉极性同一招)

烧一次板,**不重编**,只在 CCES Expressions/变量视图里改全局 **`s_m2_chmap`** 这 8 个字节切换两臂:

| 臂 | `s_m2_chmap` | 含义 |
|---|---|---|
| **A(现状/错位)** | `{0,1,2,3,4,5,6,7}` | 逐位等同今天的固件(baseline) |
| **B(修对,默认值)** | `{4,5,6,7,0,1,2,3}` | 权重落回正确物理位置 |

两臂**同一个二进制、音柱/mic/音量全不动、只差这 8 字节** → 除映射外一切可证明相同。改完主循环下一帧即生效。
> `s_m2_chmap` 是 `volatile`,下标读表时 `& (NCH−1)` 掩码 → 手误填 8~255 也不会越界读冻结表(只会绕回)。

## 3. 距离(按频点定 —— 别用死的 8m)

远场判据距离 = Fraunhofer 2L²/λ,L=0.825m:

| 频点 | 需要距离 | 说明 |
|---|---|---|
| **2kHz** | **≥8m** | ⭐**主判频点**,中带、最好定,单这一个够决定性 |
| 1kHz | ≥4m | B−A 差最大(30° 差 13dB),现象最明显 |
| 4kHz | ≥16m | 没户外/长走廊就**跳过**(8m 测 4k 仍是近场=白测) |

**没有远场空间**的两条退路:
1. **地面反射门控(gated-ground-plane)**:REW/ARTA 测脉冲响应,时间窗把**直达声在第一个反射到达前截断**——小空间也能拿到准远场极图(推荐)。
2. **相对 A/B**:在**最长可用距离**测,同距离同设置只切 `s_m2_chmap`,**只作方向性趋势证据、不作规格**(和 1m 极性 A/B 同口径)。

## 4. 信号 / 测角
- **信号**:1k、2k 的 **warble(±5%)或 1/3 倍频程窄带粉噪,读 Leq**(避纯音驻波)。只有纯音发生器就用 warble。两臂同一输入电平、同一音量。
- **测角**:0°、**±20°、±30°、±40°**、±60°、±90°(20–40° 加密 = 两版差最大的"旁瓣架");左右都测,顺带确认左右对称(镜像对驱动应保持对称,是防呆)。

## 5. 分步流程
1. `git pull`;**确认编译命令带 `-DM2_CHMAP_FIX`**。烧板,过 §1 gate。
2. 架设:音柱固定,mic 在 §3 距离、正前 0°、同高。记录**距离 / 输入电平 / 音量 / 本底 dB**。
3. **先测 B(默认值,什么都不用改)**:0→±90° 每角 Leq,@1k、@2k。
4. JTAG 把 `s_m2_chmap` 改成 `{0,1,2,3,4,5,6,7}` → **测 A**:原地重测**同样角度同样信号,别动其它任何东西**。
5. JTAG 改回默认 `{4,5,6,7,0,1,2,3}`,复测一两个角确认可复现(防漂移/防误操作)。

## 6. 判据(可证伪)
- ✅ **主判据**:**B 在 30° 比 A 低 ≥6dB**(预测 2k −7dB / 1k −13dB),0° 基本持平 → **映射修对了**,选项 B 坐实 [L1]。
- 预期同时出现:**B 的 −6dB 主瓣略宽**(2k 12.8→14.5°、1k 25.7→29.3°)——**正常,不算失败**。
- ⚠ **不拿 60°/90° 判**(巧合零点,B 可能反而抬高)。
- ❌ **若 B 在 30° 没更低甚至更差** → 假设的 **C1–C6 中间排序可能错**。转 §7 用 JTAG 活切试别的排列,不用重编。

## 7. 兜底:B 不如预期时,JTAG 试排列(不重编)
中间排序是假设,若 §6 主判据不成立:
- **保住两个实锚不动**:`s_m2_chmap[3]` = 中心那一对的权重下标(要 =7,即最大权重 1.0)、`s_m2_chmap[4]` 对应边(=0)。这两个是 Phase A 亲测的,别改。
- **先试中间几对两两对调**(如把 `s_m2_chmap` 的第 0/1、5/6 等位置互换),每次改完复测 30°,收敛到 30° 最低的那组。
- 若怎么试 30° 都不改善 → 映射不是(全部)成因,回 `BEAM_POLARITY_CLOSURE_20260708.md` §5 其它开放项(近场口径 / 自家阵列极性未验)。

## 8. 安全 / 纪律
- 默认关字节等同、`-D` 走 build **不进 `.cproject`**、**冻结 Dolph 表 / FIRA 零触碰**(只换读表下标 `g_dolph_w8_q15[perm[c]]`,表本身不动)。
- **仅 BROADSIDE v1 有效**:v2 若加逐路 frac-delay 聚焦,延迟也随物理位置变,须一并 permute,单纯换权重下标不再够。
- 功放最小起、逐步升;测完 `s_m2_chmap` 还原默认。
- **这是假设 + 本 rig 特定**;**远场 A/B 是唯一坐实门,别拿 1m 近场当结论**。
