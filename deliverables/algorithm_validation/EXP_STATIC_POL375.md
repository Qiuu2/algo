# 375Hz 声学极性测试(软件法)— 测竞品喇叭正负极有没有接反

> **今天的 rig**:我们的板 + **我们自己的功放** + **竞品的 16 只喇叭(我们接的、不知竞品正负约定)**。怀疑:喇叭 +/− 接混 → 波束被打散成"平"。
> **原理**:等幅 375Hz 单音,一次只放**选中的通道**(缓冲掩码),两两比对——**同极性相加(~+6dB)/ 反极性相消(深零点)**。单只喇叭量分贝看不出极性(取反不改幅度),**只有靠叠加相消才能显极性**,这就是本测存在的理由。
> **性质**:诊断固件改动(宏 `M2_STXT_LOCALIZE` 门控、可逆、不碰冻结 FIRA/算法)+ 台前声学测量。**CTO-gated**。
> **来源**:PM,2026-07-07。波形 python 双轨核准。**独立 critic(claude-opus-4-8)过门**:代码/波形全绿;修 1 BLOCKER + 3 MAJOR + 4 MINOR(见文末)。

---

## ⚠ 0. 先读:这个测能测到什么、测不到什么(分辨率边界)

**控制粒度 = 8 个通道(掩码只能控通道,控不到单只喇叭)。今天 16 只喇叭接到 8 路 = 每路约 2 只(具体接法未知)。** 所以:

| 情形 | 本测能否发现 | 怎么处理 |
|---|---|---|
| **跨通道**反极性(A 路整体 vs B 路整体反) | ✅ 能(Phase B 相消) | 调换该路**喇叭 +/− 端子** |
| **通道内**反极性(某路的 2 只喇叭里有 1 只反) | ⚠ 只显示为**该路 solo 变弱/趋近零**(2 只近距反相=自抵消),**说不出是哪一只** | 对该路的 2 只喇叭做**电池纸盆测试**逐只定 |
| 某只喇叭死/断 | solo 变弱(与"通道内反极性"表现相似) | 电池测试/换线区分 |

**判据边界(BLOCKER 级,必须记住)**:
- 本测**结论只到通道级**。
- **"8 路都一致、且无弱路" ≈ 通道级极性没问题**,但**不能 100% 排除"某路内部一只喇叭反接"**(会被自抵消藏住)。
- 因此:**任何一路 solo 读到弱/趋零,或统一通道极性后仍平 → 必须对相关喇叭逐只做电池纸盆测试**(见 §7),那才是逐只极性的定论。

**本测的正确定位**:先在**不拆线**的情况下,快速判"跨通道极性有没有混" + "有没有弱路";逐只极性交给电池测试收尾。

## 1. build 组合与上板前提
- **`M2_FIRA_INLOOP=1` 且 `M2_STATIC_TXTEST=1` 且 `M2_STXT_LOCALIZE=1`** 三个一起(build GUI 里 `-D` 加,**不进 `.cproject`**)。先 **clean build** 过编译再烧。
- 宏 `M2_STXT_LOCALIZE` 不定义 = 预处理输出与已提交版**逐 token 相同**(git diff 已核:原 2250 fill 完整保留在 `#else`)。
- **上板 gate**:`g_m1_valid==1` / `g_m2_setup_rc==0` / `g_m1_rx_block_count` 在增长 / `g_m1_fg_stream_live==1`。
  **⚠ `g_m2_fg_beam_live=-99`、`g_m2_out_nonzero=0` 是静态模式正常"诚实死"(poll 没跑),别当失败。**
- 阵列**横放**、broadside 正前。

## 2. 怎么改掩码(不用重编)
CCES 调试器 Expressions/变量视图里,把全局 **`g_stxt_ch_mask`** 改成下表 hex,主循环下一圈(`m2_stxt_poll_mask`)会**立即重填一次**(改前有一次瞬态,无害;不改就静止)。

| 要放的通道 | `g_stxt_ch_mask` |
|---|---|
| solo c0 / c1 / c2 / c3 | 0x01 / 0x02 / 0x04 / 0x08 |
| solo c4 / c5 / c6 / c7 | 0x10 / 0x20 / 0x40 / 0x80 |
| 参考(居中路 cR)+ 测试 cK 一对 | `(1<<R)\|(1<<K)`,例 cR=c3(0x08)时 c3+c0=0x09、c3+c7=0x88 |
| 全开(格式门用) | 0xFF |

> **参考路 cR = Phase A 测到的"物理居中"那一路,不是固定 c7**。通道号↔物理位置本就没验过(实测发现中间常是 c3=0x08,不是 c7);居中路离各路最近→相消零点最深最稳。

## 3. 回读格式门(全开 mask=0xFF,防"填错=假结论")
JTAG 回读 `s_m1_tx_buf[0]`,**同一帧内 8 路应完全相等**(等幅,不是 Dolph):

| 位置 | 期望值(8 路全同) |
|---|---|
| `s_m1_tx_buf[0]` 帧8 (idx 64–71) | 0x0c3ef153 |
| `s_m1_tx_buf[0]` 帧16 (idx 128–135) | 0x16a09e66 |
| `s_m1_tx_buf[1]` 帧16 (idx 128–135) | 0xe95f619a |

任一不符 → fill/build 错,停,别测。

## 4. Phase A — 逐路 solo(信噪比 + 无弱路 + 映射)
先测**本底噪声**(功放不发声,今天记的是 40dB [L1])。功放**最小起、逐步升**到 solo 一路时 SPL **高出本底 ≥~15dB**(否则贴本底会把后面的相消判糊)。
对 c=0..7,依次设 `g_stxt_ch_mask=(1<<c)`,每次:
1. **听/近场麦定位**:哪几只喇叭在响(**可能 1 只、也可能 2 只**——每路约 2 只喇叭,别以为掩码坏了),记 **通道 c → 物理喇叭位置**;
2. 记该路 **SPL**(前方中心,固定点)。

**判**:
- 8 路 SPL 大致齐、都高出本底 ≥15dB → 正常,进 Phase B;
- **某路明显弱 / 趋近零** = **极性候选**:很可能是**该路的 2 只喇叭内部反接自抵消**(近距反相=偶极子,正前几乎无声),**也可能**是死/断路。**别记成"板级死路"**——先标记,§7 用电池测试对该路 2 只喇叭逐只定。

## 5. Phase B — 成对极性(参考 c7,判跨通道反没反)
选 Phase A 里**位置居中、响且不弱**的通道当参考 **cR**(通道号↔位置没验过,**别默认 c7**;实测中间常是 **c3=0x08** → 就用 0x08;居中参考离各路最近→零点最深最稳)。对每个测试通道 K(除 cR 外的 7 路):
1. **麦放"cR 的喇叭"与"cK 的喇叭"两组的正中垂线**(等距点,正前约 0.3–0.5m;若每路是"对称一对"喇叭、cR 又居中,则**正前中心轴**天然等距,最干净);**这一对的 3 次测量麦不动**;
2. `mask=(1<<R)`(cR solo) 记 **L_ref**;
3. `mask=(1<<K)`(cK solo) 记 **L_K**;
4. `mask=(1<<R)|(1<<K)`(两路) 记 **L_both**。

**判**(每点取 3 次中值,近场散斑要可重复):
- **同极性**:L_both 明显**变响**(比 L_ref、L_K 都高,趋近 +6dB);
- **反极性**:L_both **远低于** L_ref、L_K(**深坑/相消**,预期低约 10–20dB [L3 期望值,非实测];挪麦一点能找到更深的坑);
- **cK 弱/死**(Phase A 已标):L_K≈本底、L_both≈L_ref(无变化)——**与"相消"区分靠 L_K 本身有没有量**。

**几何注意(critic F4)**:"相消稳健"建立在**参考与被测的喇叭彼此靠近**时最稳——所以**参考取居中路**(离各路最近)。若两者仍隔得远(375Hz λ≈915mm [L3],阵长 825mm≈0.9λ),深坑会变**尖、对麦位置敏感** → 这时**挪麦找趋势**(相消处一定明显偏低),判不清就**别硬判,转 §7 电池逐只测**。

记录每路相对 cR 的极性(SAME / OPPOSITE)。**OPPOSITE 的那些通道 = 跨通道反接组。**

## 6. 判定 → 修(单端:没有 OUTxN,只能改喇叭端)
- 8 路落成两个极性组。波束要全同极性 → 把**少数组**物理**调换该路喇叭的 +/− 端子**。
  **⚠ 本板单端输出(只用 DAC1P–DAC8P、没接 DAC1N–DAC8N)→ 没有 N 脚可换,DAC/走线侧翻不了极性;唯一能翻声学极性的点就是喇叭引线(或某级反相放大)。别去找不存在的 OUTxN。**
- 修完 → 跑 `EXP_STATIC_TXTEST.md`(2250 静态)复测:应从"平"变**指向**(±90° 落差回到十几 dB)→ 若指向恢复=今天的平确是喇叭接混,收口。
- **若统一通道极性后仍平** → 通道级极性不是(全部)原因 → 回 §0 边界:**对每路 2 只喇叭做电池逐只测**(可能是通道内单只反接),仍平再查板/输出(死弱路、去相干)。

## 7. 逐只定论:电池纸盆测试(per-speaker,收尾/兜底)
凡 Phase A 标弱的通道、或统一后仍平,对相关喇叭**逐只**做:
1. 喇叭**从功放断开**(功放关);
2. 一节 **1.5V 电池**两极**瞬间轻触**喇叭两端子(碰一下就松);
3. 看纸盆**外弹(朝你)/内缩**;约定"电池 + 碰到、纸盆外弹 → 那端是喇叭 +";
4. **16 只全按同一约定接功放**(所有喇叭 + 接功放 +),绝对无所谓、**一致即可**。
> 小喇叭 1.5V 位移可能看不清 → 换 **9V 电池极短瞬触**(更明显、别久按)。这是逐只极性的**定论**方法(不受声学分辨率限制)。

## 8. 安全 / 合规
- **连续 375Hz 低频纯音更烤喇叭、大冲程**:功放最小起、短促测、别久驻;−12dBFS 数字 + 温和功放增益足够(上次 104dB [L1] 有富余;但小喇叭 375Hz 效率可能比 2250Hz 低 10–20dB,solo 若贴近 40dB 本底就先把增益抬到 §4 的 ≥15dB 再测)。
- **⚠ 相消零点处纸盆仍是满冲程**(相消是声学抵消、不是电停):**别为了"听见零点"猛加增益**——喇叭在你听到"安静"时其实在满幅摆动,易过冲/过热。
- 不碰 `.cproject`/`.project`;诊断 build 仅 `-D`,**不提交主线默认**;冻结 FIRA/算法零触碰(fill 只写非冻结 TU 的 `s_m1_tx_buf`)。
- 下标已核:`63*8+7=511<512`,无越界。宏默认 OFF,测完还原=字节复原。
