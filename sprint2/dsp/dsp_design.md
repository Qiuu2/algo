## 1. 系统架构概述

### 1.1 基本参数

| 参数 | 值 |
|---|---|
| DSP 芯片 | ADSP-21569 SHARC+ |
| 核心时钟 | 1 GHz（保守算力口径 1.5 GFLOPS） |
| 片上 SRAM | L1 640 KB + L2 1024 KB = 1664 KB |
| 阵元数量 | 16 |
| 阵元间距 | 55 mm[L1拆机实测]（DEC-S3-GEOM-01；旧 30mm 系视觉估测错误，已作废） |
| 声速 | 343 m/s |
| 采样率 | **48 kHz** |
| 帧大小 | 64 samples（1.33 ms） |
| 子带数量 | 4（SB0~SB3） |

### 1.2 采样率选择：48kHz

选择 48kHz 的理由：

1. **频率覆盖充足**：最高指向频率 8kHz，奈奎斯特频率 24kHz，留有 3× 余量。
2. **生态兼容性**：与 PCM512x、CS4272、AD1938 等 TDM Codec 无缝对接，无需额外异步采样率转换（ASRC）。
3. **延迟经济**：64 samples @ 48kHz = 1.33ms，满足广播类应用 <5ms 延迟预算。
4. **无需 96kHz**：产品目标为博物馆/广播，不需要 Hi-Fi 指标，48kHz 完全够用。

### 1.3 信号流程总览

```
  ┌──────────────────────────────────────────────────────────────────┐
  │            1 通道节目源输入 (mono program, 48kHz)                  │
  │            (单声道音频，经 TDM/I2S 输入)                            │
  └──────┬───────────────────────────────────────────────────────────┘
         │ DMA Ping-Pong (64 samples × 1ch × 32bit)
         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │          分析滤波器组 (dyadic 树形, 1 通道 → 4 子带)                │
  │  SB0: 500-1kHz  ÷16→3kHz   SB2: 2k-4kHz   ÷4→12kHz            │
  │  SB1: 1k-2kHz   ÷8→6kHz    SB3: 4k-8kHz   ÷2→24kHz            │
  └──────┬───────────────────────────────────────────────────────────┘
         │ 4 子带信号 (单路, 低速率)
         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │      通道扇出 + 加权 + 延迟 (per 子带: 1 通道 → 16 通道)            │
  │  对每个输出通道 m = 0..15 (阵元):                                   │
  │    · Dolph-Chebyshev 加权 w_k[m]  (预计算 Q15)                    │
  │    · 整数延迟(环形缓冲≤44smp) + 8阶分数延迟FIR (波束指向 τ_m)       │
  └──────┬───────────────────────────────────────────────────────────┘
         │ 每子带 16 路 加权/延迟后的信号
         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │       合成滤波器组 (per 通道: 4 子带 → 全带, ×M 增益补偿)           │
  │       16 个输出通道各自重建为全带信号                                │
  └──────┬───────────────────────────────────────────────────────────┘
         │ 16 通道全带输出 (48kHz)
         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │           后处理 (可选: per-ch DRC / EQ / 电平 / 限幅)             │
  └──────┬───────────────────────────────────────────────────────────┘
         │ DMA Ping-Pong 输出 (16ch)
         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │     16 通道 TDM 输出 → Codec(DAC) → D类功放 → 16 单元线阵         │
  └──────────────────────────────────────────────────────────────────┘
```

> **方向说明（修订记录）**：本图为**发射（扬声器）阵列**的物理实现方向——
> **1 路节目源 → 分子带 → 各通道加权+延迟扇出 → 合成 → 16 路输出 → 阵列**。
> 旧版误用接收（麦克风）阵列的"16→1 加权求和"画法（已更正）。
>
> **算法对称性**：由阵列互易性，发射波束权重 w_k[m] 与延迟 τ_m 的**数值**与
> 接收波束形成完全相同（同一组 Dolph-Chebyshev 系数 + 同一组指向时延），
> 故 §3/§5 的系数计算、算力(6.48 MMAC/s 核心)、SRAM 估算**均不受影响**；
> 仅数据流方向相反（扇出 vs 求和）。实现上：接收是"16 路输入累加成 1 路"，
> 发射是"1 路输入复制为 16 路后各自加权延迟"，MAC 次数同阶。

## 2. 子带划分方案

### 2.1 子带参数表

| 子带 | 频率范围 | 抽取率 M | 子带率 fs/M | 原型 FIR 阶数 N | d/λ (低端) | d/λ (高端) | 栅瓣 |
|------|----------|---------|------------|----------------|-----------|-----------|------|
> d/λ 列按 **d=55mm[L1拆机实测]**（DEC-S3-GEOM-01）重算（旧 d=30mm 已作废）。

| 子带 | 频率范围 | 抽取率 M | 子带率 fs/M | 原型 FIR 阶数 N | d/λ (低端) | d/λ (高端) | 栅瓣 |
|------|----------|---------|------------|----------------|-----------|-----------|------|
| SB0 | 500–1000 Hz | 16× | 3000 Hz | 31 | 0.080 | 0.160 | 无 |
| SB1 | 1000–2000 Hz | 8× | 6000 Hz | 63 | 0.160 | 0.321 | 无 |
| SB2 | 2000–4000 Hz | 4× | 12000 Hz | 63 | 0.321 | 0.641 | 无 |
| SB3 | 4000–8000 Hz | 2× | 24000 Hz | 63 | 0.641 | 1.283 | **6.2–8kHz 有栅瓣** |

**注**（d=55mm）: d/λ_max = 0.055 × 8000 / 343 = 1.283 > 1.0。**500Hz–5kHz 安全**（d/λ<1.0）；**6.2–8kHz 出现栅瓣**[L1/几何]——半波长判据 3118Hz，严格判据(broadside, d=λ) 6236Hz，6.24kHz 起栅瓣峰 0dB（与主瓣等高）、带外泄漏 78–89%[L2 仿真]。应对：强指向规格上限由 8kHz 降级为 6kHz(对内)/5kHz(对外保守)（方向1，CTO 采纳），SB3 上半段落在降级规格之外。**栅瓣变化不影响算力**（算力与 d 无关）。

### 2.2 子带划分理由

**SB0（500–1kHz，宽波束/补声频段）**
- d/λ < 0.088，自然形成宽波束，无强指向性
- CTO 确认 ≥1kHz 才需要强指向，SB0 可做全向播放（舒适声场填充）
- 抽取 16× 后算力最低（3kHz 速率），用较低阶 31 阶原型即可
- 16× 抽取允许多相 FIR 只需约 2 阶等效每个多相分支

**SB1（1k–2kHz，关键指向频段）**
- 声学规格要求 ≥1kHz 强指向，SB1 是最关键子带
- d/λ = 0.087–0.175，波束宽度约 20°–40°，满足 30° 目标
- 8× 抽取在 6kHz 速率下精确处理，63 阶原型保证 -60dB 阻带
- 分数延迟精度需求最高（波束指向精度来自此子带）

**SB2（2k–4kHz，高频指向）**
- d/λ = 0.175–0.350，指向性良好，波束更窄（约 10°–20°）
- 4× 抽取，12kHz 速率，63 阶原型
- 同一波束指向角在此频段更精确

**SB3（4k–8kHz，超高频指向）**
- d/λ = 0.35–0.70，极强指向性，波束宽度 <10°
- 2× 抽取，24kHz 速率，63 阶原型
- 高频端频率扩展至 8kHz（音乐内容摩擦音、辅音增益）

### 2.3 抽取策略优势

通过多相滤波器分解，FIR 原型滤波器（全速率 N 阶）在子带速率 fs/M 下运行，
等效算力为全速率方案的 1/M：

```
全速率方案: (N+1) × N_ch × fs     MACs/s  (超过 1000 MMAC/s)
多相方案:   (N+1) × N_ch × (fs/M) MACs/s  (仅约 44 MMAC/s)
```

## 3. 分析滤波器组设计

### 3.1 多相 FIR 原型滤波器规格

| 参数 | SB0 | SB1–SB3 |
|------|-----|---------|
| 滤波器类型 | 线性相位 FIR（等波纹） | 线性相位 FIR（等波纹） |
| 通带截止 | 1kHz（-0.5dB） | 各子带上沿（-0.5dB） |
| 阻带衰减 | ≥ 40 dB | ≥ 60 dB |
| 原型阶数 N | 31 | 63 |
| 多相分支数 | 16 | 4 / 8 |
| 每分支阶数 | 2 | 8 / 16 |

### 3.2 带通实现方式

子带 SB_k 的带通滤波器通过将低通原型 h_LP(n) 调制到子带中心频率实现：

```
h_BP_k(n) = h_LP(n) · exp(j·2π·fc_k·n/fs)
```

其中 fc_k 为各子带中心频率（707 / 1414 / 2828 / 5657 Hz）。

实现时取实部（余弦调制），得到线性相位实系数带通 FIR。

### 3.3 加权系数实现

各子带的波束指向权重 w_k[m]（m 为通道序号）：

```
w_k[m] = exp(-j · 2π · fc_k · τ_m / 1)
```

其中 τ_m = m·d·sin(θ_steering)/c 为第 m 通道的理论时延。

对于 DAS（延迟求和）波束形成，权重等幅 |w_k[m]| = 1/N_CH，相位补偿时延。
这些权重预计算并存为 Q15 复数向量（实部+虚部各 Q15）。

## 4. 分数延迟滤波器

### 4.1 设计指标

| 参数 | 值 |
|---|---|
| 最大波束偏转角 | ±45° |
| 最大总时延 | (N_CH-1)×d×sin(45°)/c = 0.928 ms = 44.5 samples@48kHz |
| 整数延迟部分 | 0–44 samples（环形缓冲区实现，零算力） |
| 最大分数延迟 | 0.0–0.99 samples |
| 分数延迟 FIR 阶数 | **8 阶**（8 抽头） |
| 插值方法 | sinc-windowed Kaiser 窗（β=5.65） |
| 频率平坦度 | <0.5dB，0–0.45×fs_sub |

### 4.2 sinc-windowed Kaiser 设计

对于分数延迟 d（0 ≤ d < 1），8 阶 FIR 系数为：

```
h[n] = sinc(n - d - 3.5) · w_Kaiser[n],  n = 0,1,...,7
```

其中：
- 中心偏移 3.5（8 抽头中心）
- Kaiser 窗参数 β = 5.65（对应旁瓣 ≈ -40dB）

### 4.3 实现架构

```
整数延迟（环形缓冲，每通道）
x_in →  [ 环形缓冲区, 深度 = max_int_delay + 8 ] → x_delayed
                                                           |
                                               分数延迟 FIR（8阶）
                                                           |
                                                       x_compensated
```

内存分配：
- 环形缓冲区：16 通道 × 54 采样 × 4 字节 = 3,456 字节
- 分数延迟系数：4 子带 × 16 通道 × 8 阶 × 2 字节（Q15） = 1,024 字节（可复用）
- 分数延迟状态：4 子带 × 16 通道 × 8 状态 × 4 字节（Q31） = 2,048 字节

### 4.4 波束指向更新频率

分数延迟系数（对应波束指向角 θ）在更新波束时重新计算：
- 固定波束（出厂预设）：系数固化在 flash，启动时载入
- 可控波束（扩展功能）：系数在非实时上下文中重新计算，原子切换（双缓冲）

### 7.3 DMA Ping-Pong 方案

```
缓冲 A (ping):  当前帧 → DSP 正在处理
缓冲 B (pong):  下一帧 → DMA 正在写入

FSYNC 边沿时:
  1. DMA 切换：B→A（新数据变为当前帧）
  2. 触发中断：通知 DSP 处理完成的 A
  3. DSP 开始处理新的 A，同时 DMA 写 B

约束: DSP 处理时间 < T_frame = 1.33ms
实际估计: 58.1 MMAC/s / 1500 MMAC/s × 1.33ms = 0.052ms << 1.33ms  (裕量 26×)
```

### 8.2 MAC 运算精度

- 分数延迟：< 0.3 dB（8 阶插值频率平坦度）

## 9. 技术风险

### 风险 1（最大风险）：SB0 与 SB1 交叉过渡带相位一致性

**描述**：500Hz–1kHz 子带边界，SB0（宽波束）和 SB1（强指向）的相位必须连续。
多相 FIR 的群延迟（group delay）在过渡带如果不一致，
合成时会产生梳状滤波效应（频谱凹陷），影响 1kHz 附近的声像自然度。

**缓解**：两个子带的原型 FIR 保持严格线性相位（对称系数），合成时引入等群延迟补偿。
实测验证：Python 仿真检查 500–1500Hz 频段的合成幅频响应，要求平坦度 ±1dB。

### 风险 2：分数延迟精度对 1kHz 指向的影响

**描述**：8 阶分数延迟 FIR 对 8kHz 以下的频响约 -0.5dB 误差，
但在 1kHz 处对应波束指向精度约 ±0.5°（d/λ=0.088 时误差敏感度较低）。
若分数延迟系数量化误差较大（粗量化 Q15），高频 SB3 的旁瓣可能抬高 2–3dB。

**缓解**：SB3 的分数延迟系数用 Q31 存储，提升精度。验收：Python 仿真旁瓣 < -15dB。

### 风险 3：SB3 实时性（最高子带速率 24kHz）

**描述**：SB3 在 24kHz 速率下运行，单帧为 64/2 = 32 samples，
帧时间仅 0.67ms，中断响应要求更紧。若 DSP 进入低功耗或缓存未命中，
可能导致 DMA Underrun。

**缓解**：所有热路径数据置于 L1 SRAM（零等待访问），关闭非必要中断，
SPORT DMA 使用最高优先级中断（IRQ 优先级 = 0）。

### 风险 4（次要）：BCLK 24.576MHz 时钟精度

**描述**：24.576MHz = 48kHz × 16 × 32，需要 PLL 精确分频。
ADSP-21569 的 PLL 支持整数/分数分频，但分数分频会引入微小抖动（jitter）。
对音频系统，抖动 > 200ps 可能影响 Codec THD+N。

**缓解**：优先从外部晶振（24.576MHz TCXO）直接驱动 BCLK，DSP PLL 仅用于核心频率。

### 关键认知：多速率树形群延迟折算到输入率

分析树第 i 级运行在速率 fs/2^(i-1)，其半带 FIR 群延迟 = (Ntap−1)/2 个【该级速率】样点，
折算为时间 = (Ntap−1)/2 / (fs/2^(i-1))。**越低速率的级，单位抽头贡献的时间越大。**
scipy 核实：63 抽头半带 FIR 群延迟实测 31.0 samples = 理论 (63−1)/2 ✓。
