## 1. 推荐的最终阵列方案

> 这是 PM 的工程推荐；**最终 N、d 由你在 Gate 1 拍板**（见第 5 节）。

**主推荐：N = 24，d = 30 mm，Dolph-Chebyshev −30 dB 加权**（阵长 690 mm）

| 维度 | 主推荐 N=24/d=30/Dolph-30 | 经济备选 N=16/d=30/uniform |
|------|--------------------------|---------------------------|
| -6dB 波束宽度 @2kHz | ≈ 18–21°（达标 ≤30°） | ≈ 22–25°（达标） |
| 峰值旁瓣级 | **−30 dB** | −13 dB |
| 阵长 | 690 mm | 450 mm |
| 适配分区广播 | 优（旁瓣低，邻区串扰小） | 一般（旁瓣高，易溢出邻区） |
| I/O 复杂度 | 需 2×TDM / Codec×3 / 功放×12 | 单 TDM / Codec×1 / 功放×8 |
| 整机料成本（估，1k 量级） | ≈ ¥1,220–2,350 | ≈ ¥940–1,785 |

**推荐 N=24 的理由**：① 竞品反推为 N≈20，要在指向性上对标必须 ≥20 元；② 博物馆/商场**分区广播**场景对旁瓣极敏感，−30 dB 旁瓣显著降低邻区串扰，而 N=16 受阵元数限制，要么用重加权导致波束过宽超标、要么用均匀加权只能拿到 −13 dB 旁瓣；③ DSP 侧已确认 24ch 仅 83 MMAC/s（裕量 18×），**算法不是约束**，唯一代价是 I/O 与约 ¥450 BOM 增量。

**推荐 d=30 mm 的理由**：栅瓣安全。d=35mm 在 4.9kHz 即触及 d/λ=0.5 转向栅瓣临界、8kHz 出现 ±37.8° 栅瓣；d=30mm 栅瓣推迟到 ~5.7kHz，且与 Gate-1 基线一致。代价是阵略短、波束略宽（约 ×1.17），但仍远在 30° 规格内。

## 2. 完整 DSP 算法链路（ADSP-21569）

```
16/24ch TDM-in (48kHz,32bit)
   │  BCLK 24.576MHz · DMA ping-pong (64 smp/buf)
   ▼
[每通道] dyadic 树形半带分析滤波器组  ── 4 子带
   ├ SB0 500–1k Hz   (M=16, 宽波束补声, d/λ<0.09 不做强指向)
   ├ SB1 1k–2k Hz    (M=8,  ★关键指向子带, CTO 要求 ≥1kHz)
   ├ SB2 2k–4k Hz    (M=4,  高频指向)
   └ SB3 4k–8k Hz    (M=2,  超高频, d=30mm 此带须监控旁瓣)
   ▼
[每子带·每通道] Dolph-Chebyshev 加权 × 分数延迟 FIR(波束指向/聚焦)
   ▼
[每子带] 跨通道求和 (Delay-and-Sum 核心)
   ▼
合成滤波器组 (×M 增益补偿, 互补差分低通组, 重建平坦 0.004dB)
   ▼
TDM-out → Codec → D类功放 → 阵列
```

**实测/核算指标**（真实 Python+C 验证，非纸面）：

| 项 | 值 | 余量 |
|----|-----|------|
| 子带 FIR 真实阶数 | 437 抽头（scipy Kaiser），阻带 58–73 dB | — |
| **全速率实现** | 1480 MMAC/s | **仅 1.0×（不可行）** |
| **dyadic 树形实现** | **56.4 MMAC/s** | **27×** ✓ |
| 核心波束形成 | 6.48 MMAC/s | 15×（vs <100 目标）✓ |
| SRAM 占用 | 27.3 KB / 1664 KB | 1.6% ✓ |
| 采样率 | 48 kHz | — |
| 重建幅频平坦度 | ±0.004 dB (0.5–7kHz) | ✓ |
| 系统群延迟 | **4.54 ms** | ⚠️ 逼近广播 <5ms 预算 |

CCES 项目骨架已生成（config.h / fir.c+h / beamformer.c+h / main.c / fir_coeffs.h 真实系数）。
⚠️ **关键缺口**：当前 C 是**全速率参考实现（1.0× 裕量）**，27× 裕量依赖的 **dyadic 树形结构尚未写成 C** → Sprint 3 首要任务。

| D类功放 | TAS5825M ×8 | ×12 | — |
| 电源（明纬+Buck+LDO） | — | — | 24ch 功率档需提升 |
| 时钟 24.576MHz TCXO | — | — | +缓冲分配 |
| 扬声器单元 | ×16 | ×24 | ¥15–35/只，价格弹性最大 |
| PCB/连接器/辅料 | — | — | — |
| **整机料成本（修正后）** | **≈ ¥940–1,785** | **≈ ¥1,220–2,350** | 已扣减 DSP 高估 |

**功耗**：待机 16ch~4W / 24ch~7W；典型语音(-6dB) 16ch~54W / 24ch~80W；最大正弦 208 / 310W。

**成本陷阱**：① DSP 单点成本+供货风险（无 pin-to-pin 替代，建议锁货源）；② 扬声器单元单价弹性大，量产前必须锁供应商。

## 4. 与竞品的性能差距分析

竞品反推：**N≈20 / d=35mm / Dolph-Chebyshev −20dB / 阵长≈665mm**（最小二乘 RMS 2.84dB，置信度中等；4kHz 仅供参考）。

| 指标 | 竞品（实测/反推） | 我方主推荐 N=24 | 差距判断 |
|------|------------------|----------------|----------|
| 前后比 @1k/2k/4k | 19.3 / 18.3 / 22.5 dB | 设计目标对齐（仿真模型不预测真实后向辐射，需实测验证） | 实测后定论 |
| -6dB 波束宽度 @2kHz | 模型隐含 ≈18.1°（实测仅 4 角度，无法可靠测） | ≈18–21° | **基本持平** |
| 峰值旁瓣 | Dolph-20dB 类（约 −20dB） | **−30dB（更优）** | **我方领先** |
| 轴向 SPL/灵敏度 @0° | **106–111 dB**（高灵敏度） | **未知** | ⚠️ **重大未知差距** |
| 低频(<500Hz)指向 | 弱（前后比仅 4dB） | 同样不做（与 ≥1kHz 策略一致） | 持平 |

**诚实结论**：
- **指向性维度我方有竞争力**（波束宽度持平、旁瓣更优）。
- ⚠️ **绝对声压/灵敏度是当前最大盲区**：我方仿真为归一化方向图，**不能与竞品绝对 SPL 直接比**；竞品 0° 106–111dB 属高灵敏度产品，我方尚未做扬声器单元选型与最大 SPL 设计 → 这是 Sprint 3 必须补的对标维度。
- 竞品反推为参考级证据（4 角度采样、点声源假设、4kHz 误差±2–3dB），**不可当作精确逆向**。

## 5. 进入 Sprint 3 之前需要你（CTO）决策的事项

**Gate 1 决策（阻塞 Sprint 3，必须拍板）：**

1. **🔴 阵元间距 d**：30mm（栅瓣安全，PM+Critic 推荐）vs 35mm（指向略好但 8kHz 栅瓣）。
2. **🔴 阵元数 N**：24（对标竞品、旁瓣-30dB、分区广播优，+¥450 且需 2×TDM/Codec×3/功放×12）vs 16（成本最低、结构简单，但旁瓣仅-13dB）。
   - ⚠️ 此决策同时锁定硬件 I/O 架构（单 TDM vs 双 TDM），不可拖。
3. **专利动作**：是否启动差异化专利申请？候选 5 方向——①子带频率不变波束 ②低频差分+高频相控混合 ③单元在线自校准 ④近场多区聚焦 ⑤AI 辅助声场自适应。
4. **专利核查委托**：是否委托代理在 CNIPA 核查 **broadside 可控阵列组合专利（Fincham 类，约 2027 到期）中国同族**？（注：Critic 已纠正——竞品参考的 Boone endfire 专利 EP1986464A1 对我方 broadside 路线**适用性低，原报告过度告警**；真正风险在可控阵列组合专利。）

**需你知悉的技术风险（非决策，但 Sprint 3 须跟踪）：**

5. **算力达标依赖未落地的重构**：27× 裕量基于 dyadic 树形结构，但当前 C 仅全速率参考实现（1.0× 裕量）→ **Sprint 3 第一优先级**是把树形结构写成 C 并在 EZKIT 实测。
6. **群延迟 4.54ms** 逼近广播 <5ms 预算，需确认是否可接受或优化滤波器阶数。
7. **绝对 SPL 盲区**：需在 Sprint 3 选定扬声器单元并给出灵敏度/最大 SPL 目标，才能完成对竞品的完整对标。
