## 0. 为什么有这条制度（背景）

DOC-AUDIT-SIM-001 仿真审计暴露了一个**系统性流程缺陷**：纸面/仿真结论被措辞成"实测"，写进 `decisions_log` 当成不可逆决策依据。

**典型病例 PF-1**：算力裕量 27×/49× 是**纸面解析值**（树形结构当时从未落 C），却被当成"已验证"写进 DEC-S1-004，**LOCKED 了芯片选型并触发了不可逆采购**。后经 P0-2 树形 C 实算，真实裕量是 17×/33×（纸面低估 ~1.6×）。若当初据 49× 草率降级到更省的芯片，可能误判。

其它病例：群延迟 12.53ms 标"实测"实为 scipy 仿真（PF-2）；SPL 117dB 全占位无 BL（PF-3）。

**典型病例 PF-8（v1.2 新增，"估测当实测"+ 冲突未重审）**：阵元间距 **d=30mm** 是 Sprint 2 **视觉/拍脑袋目测值**（无任何卡尺测量或工具支撑），却被 LOCKED 进 DEC-S2-006 作为阵列几何基线，污染规格上限与工装尺寸。Sprint 3 拆机用游标卡尺**实测真值 d=55mm/L=825mm**，与 LOCKED 的 d=30 显著冲突（孔径差近 1.83×），但**未触发对 LOCKED 决策的强制重审**，反而以"两条几何并存"被合理化掩盖，直到暴露才以 DEC-S3-GEOM-01 撤销 d=30。PF-8 与教训①半角误读、②纸面当实测、③假性裕量**同根——数字没有来源身份证**，且额外暴露两个新缺口：（a）比 L4 更低的"纯目测"层级缺位（d=30 是目测，连行业参考值都不是，却作了 LOCKED 依据）；（b）实测与 LOCKED 冲突时无强制重审机制。本制度 v1.2 据此增设 **L0 等级**、**L1↔LOCKED 冲突强制重审规则** 与 **Critic C6 几何门禁**。

**根因**：数字没有"来源身份证"，读者无从分辨一个数是测出来的、算出来的、还是拍脑袋的，于是低可信度的数被当成高可信度用。本制度强制给每个进入决策的数字标注来源等级，并按等级限制其决策权重。

**典型病例 LESSON-012（v1.4 缘起，"单工具笔误量级偏差"）**：TASK-MATLAB-RV 三轨验证（numpy/scipy + acoustic teammate MATLAB R2026a + CTO MATLAB）中，**M1 WNG 草稿 bug**（公式多除 |a|²=N，致约 **12dB 量级**偏差，误差恰 = 10log₁₀(16)=12.041dB）在落盘前被三轨交叉核**当场抓出、并精确定位为"numpy 草稿错、MATLAB/CTO 对"**。若单轨开发，该 12dB 量级笔误将直接落盘污染 LOCKED。此即"关键数字须第二独立工具交叉核"的硬证据，本制度 v1.4 据此增设 **铁律七 双轨独立工具核 + C5 扩**。

**典型病例 LESSON-013（v1.5 缘起，"已入手未入库"盲区）**：`定向音柱AI数据.docx` 硬件团队于 **2026-05-28** 发给 CTO，CTO 于 **2026-05-30** 才转交 PM / agent team 进 KB——间隔约 **2 天，超 24h**。这两天间隔内 Sprint 3 处于活跃产出期（三轨验证、仿真全景、标准 12 点合规等均在跑），**全部未用上该 docx 的信息**（典型后果：R3 SPL 仍以占位值流通，未被 docx 中可能存在的硬件实测/datasheet 数据替换）。根因：**CTO 层外部信息流无 24h 强制入库机制**——信息「已入手（到了 CTO 手里）」但「未入库（没进 KB / 没同步 handover）」，下游 agent team 看不见，等于没收到。本制度 v1.5 据此增设 **铁律六 信息透明义务 + Critic C8 入库传播门禁 + CTO 外部接收声明义务**。

**LESSON-013 与 PF-9 的对偶（数字生命周期两端）**：PF-9 = **已撤回未清扫**（数字该出库却赖在库里）= **出站**问题 → 铁律五 / C7 守；LESSON-013 = **已入手未入库**（数字该进库却卡在 CTO 手里）= **入站**问题 → 铁律六 / C8 守。**出站 C7 防"该走的赖着不走"，入站 C8 防"该进的进不来"**，数字生命周期两端闭合。

## 1. 五级来源定义（v1.2：在原四级前增设 L0）

| 标签 | 等级名 | 定义 | 可信度 | 示例 |
|------|--------|------|--------|------|
| **L1** | **实测** | 真实硬件/样品/环境测得，有原始数据可追溯 | 最高 | EZKIT 实测 MCPS（cycle counter）；消声室实测 BW；DATS 实测 T/S 参数；功放实测效率；**游标卡尺实测 d=55mm** |
| **L2** | **仿真** | 经验证的工具/模型算得，工具本身可信 | 高 | MATLAB Phased Array 算 BW；scipy `freqz` 验滤波器组阻带；树形 C host 重建验证 SNR |
| **L3** | **解析估算** | 公式/手算/理论模型，含简化假设 | 中 | MMAC 解析预算（MACs=(N+1)×fs_sub×N_ch）；阵列增益 20log₁₀(N)；群延迟 (tap−1)/2/fs |
| **L4** | **占位假设** | 行业参考值/拍脑袋/未验证 | 低 | 占位 T/S（BL=4.5、Re=6Ω）；假设功率 P_ch=10W；灵敏度 88dB[估算]；"-2dB 保守修正"拍脑袋 |
| **L0** | **目测/视觉估算** | **目测、照片/视觉估算、拍脑袋猜测，无任何测量或工具支撑**（连厂商 datasheet 标称都没有） | **最低** | **视觉估测 d=30mm（PF-8 病根）**；照片量外壳"约 450mm"；肉眼数阵元"大概 16 个" |

**⛔ L0 铁律（v1.2 新增，比 L4 更严）**：**L0 严禁用于任何 LOCKED 决策**——不仅是不可逆决策，**任何写入 decisions_log 的 LOCKED 状态决策（含强约束/几何锁定）都不得以 L0 为依据**。理由：L4 至少是行业参考值/datasheet 标称（有外部出处），而 L0 是纯目测、毫无测量支撑，可信度低于 L4。**PF-8 的 d=30 即典型 L0 被当 LOCKED 依据**。L0 只允许出现在「探索讨论」阶段且**不得直接进 decisions_log**；任何 L0 数欲进入设计输入，必须先升级到 L1（实测）。违者 → Critic C6 BLOCKER。

**等级判定优先级（复合量定级，v1.1 细化）**：一个数若同时含多种来源成分——
- 若**低等级成分实质决定结果**（作乘数/主导项），**整体按最弱成分定级**。例：SPL 117dB = 阵列增益(L3) + **占位灵敏度(L4) + 占位功率(L4)**，占位输入直接缩放结果 → **整体定 L4**。
- 若低等级成分仅为**次要可加项**（不主导结果），整体按**决策关键的主导成分**定级，但**必须在标注里显式点出低等级成分**。例：端到端延迟 12.53ms = scipy 群延迟(L3 主导) + DAC datasheet 0.75ms(L4 次要项) → 标 **`[L3 解析/scipy 群延迟，含 L4 datasheet DAC 项，非硬件实测]`**。

**边界案例裁定（v1.1 补，Critic F-2）**：
- **datasheet 标称值（厂商典型值、非本项目自测）= L4，子类标注「L4/datasheet 标称」**，以区别于「L4/拍脑袋」（前者比后者可信，但仍非实测，不得当 L1）。例：ADAU1966A DAC 群延迟、喇叭 datasheet 灵敏度。
- **经实测数据标定过的仿真 = 「L2+ 已校准」**（介于 L2/L1）：置信度高于纯仿真，但插值/外推区仍非实测，**仍不得当 L1 用于不可逆决策**，在强约束决策里可视同 L2 高端。
- **L2 仿真细分**：「L2/已验证工具」（如 MATLAB beamwidth、scipy freqz）与「L2/含近似」（如 cosine 阵元代理障板衍射、互耦粗代理）；后者标注须点明近似性质，本质介于 L2/L3，从严对待、不得当 L1 实测。

## 2. 标注格式（强制）

**规则**：任何进入 decisions_log / 报告结论表 / 规格的数字，其后**紧跟方括号标签**：`数值 [等级/出处简述]`。

示例（正确写法）：
- `算力裕量 33× [L2 仿真/桌面树形C MCPS]`
- `BW@1kHz 29.28° [L2 仿真/MATLAB beamwidth 全角]`
- `SPL 117dB [L4 占位/未用真实BL，理论上限]`
- `端到端延迟 12.53ms [L3 解析/scipy 群延迟，非硬件实测]`
- `T/S BL=4.5 [L4 占位/datasheet 缺，真值 DATS 送测中]`
- `竞品 BW@1k 16° [L4 占位/4点稀疏插值，已撤回]`

**最低要求**：标签至少含「等级 + 一句出处」。鼓励附数据文件路径（如 `[L2/sweep_d55_results.csv]`）。

## 3. 决策权重规则（按可逆性分档）

| 决策类型 | 定义/示例 | 允许的最低来源等级 | 约束 |
|---------|----------|------------------|------|
| **不可逆决策** | 采购、芯片选型、流片、对客户的性能承诺 | **L1**；或 **L2 + 风险声明 + 总工签字** | **L3/L4 不得作唯一依据** |
| **强约束决策** | 算法路线锁定、规格冻结 | **L2 起**；L3 须挂「待验证」标记 | L3 单独支撑须有回填计划 |
| **方向性决策** | 选型对比、参数初选、架构倾向 | **L3 可用** | 标注等级即可 |
| **探索讨论** | 头脑风暴、可行性粗估 | **L4 可用** | **但不得直接进 decisions_log** |

**关键红线**：占位（L4）和解析估算（L3）**不得作为不可逆决策（采购/选型/流片）的唯一依据**。

**🔒 可逆性升级原则（v1.1 补，Critic F-1 —— 堵"伪装降档"空子）**：
- 决策档位**不得自我降档以规避等级要求**。凡涉及**采购下单 / 流片 / 对客户书面承诺**的决策，**一律按"不可逆"处理**，不得自称"方向性/初选"来降低门槛（PF-1 本质就是芯片选型被当低门槛处理）。
- 档位归类**有争议时从严**——按更不可逆的一档判定。
- **"L2 + 风险声明 + 总工签字"豁免**不可逆决策的 L1 要求时，**签字必须记入该 DEC 条目的「验证状态」字段**（写明签字人 + 日期 + 豁免理由），否则豁免无效、Critic 按 L1 缺失判 C3 BLOCKER。

## 4. 九条铁律（v1.2：增设铁律四；v1.3：增设铁律五；v1.5：增设铁律六、铁律七；v1.6：增设铁律八；v1.7：增设铁律九）

> **铁律一**：**L4 占位值严禁作不可逆决策的唯一依据；L0 目测值严禁作任何 LOCKED 决策依据。** 采购/选型/流片若仅有 L4 支撑 → 一律 BLOCKER，必须先升级到 L1（实测）或 L2+签字；L0 见第 1 部分 L0 铁律。

> **铁律二**：**L3 解析支撑强约束决策，必须挂「待 L1/L2 验证」标记，实测到位后回填。** 例：DEC-S1-004 芯片选型当初依据 L3 解析算力，应标「待 EZKIT 实测回填」，并在实测后用 L1 值替换、记录差异。

> **铁律三**：**措辞红线——严禁把 L2/L3/L4 描述成「实测/已验证/确认」。** "实测"一词仅限 L1。仿真说"仿真/解析得出"，估算说"估算"，占位说"占位/假设"。违者 BLOCKER。

> **铁律四（v1.2 新增，缘起 PF-8）——实测与 LOCKED 冲突强制重审，禁"并存"折衷**：任何 **L1 实测值与已 LOCKED 决策冲突**时，**自动触发该 LOCKED 决策的强制重审，严禁以"并存/折衷/两条路线"合理化掩盖**。冲突发现者（任何 teammate/PM）须**立即上报 PM + CTO**，不得自行合理化。例：PF-8 拆机 L1 d=55 与 LOCKED d=30 冲突，应立即触发 DEC-S2-006 重审，而非"两条几何并存"长期流通。违者 BLOCKER。

> **铁律五（v1.3 新增，缘起 PF-9）——撤回必须全库传播，三步缺一不可**：任何数字被撤回时，撤回动作 = ① **发出撤回声明**（注明被撤值、撤回原因、撤回令号如 F-AC-01）+ ② **全库反向扫描**该数字及其派生值/别名的所有引用 + ③ **逐处加撤回警示标签或删除**。三步未全部完成，撤回**不生效**，不得对外/对下游宣称"已撤回/已修正"。违者 BLOCKER（由 Critic C7 守门）。理由：PF-9 表明，仅在源头撤回（F-AC-01 撤回 14.9°）而不反向清扫，残留"实测"标签会在权威画像文档继续喂养外部误引，造成 PF-6 同款"估测当实测"复发。

> **铁律六（v1.5 新增，缘起 LESSON-013 docx 事件）——信息透明义务，外部输入 24h 入库**：**凡 CTO 收到的外部技术输入（硬件参数表 / datasheet / 测试报告 / 标准文件 / 客户反馈等一切外部技术输入），自 CTO 收到（入手）时刻起 24h 内须进 KB（+ handover 同步）。** 责任流：CTO 收到 → 经 PM 触发归档 → PM 入 KB 并在 handover 标注来源/日期/等级。超 24h 未入库即违规（由 Critic C8 守门）。**与铁律五对偶**：铁律五守**撤回传播 = 出站**（数字该出库时改全改干净、不残留），铁律六守**入库传播 = 入站**（数字该进库时按时进、不滞留 CTO 手里）；**铁律五管"出"、铁律六管"进"**，对偶成立，数字生命周期两端均有铁律守门。

> **铁律七（v1.5 新增，缘起 LESSON-012 M1 WNG 12dB bug）——关键数字双轨独立工具核**：进入**不可逆 / 强约束决策**的关键**几何量 / 算力量 / 统计量 / 标准合规判定**数字，**须有至少第二条独立工具实现链路交叉核对**（例：numpy/scipy + MATLAB；或不同实现者 / 不同随机种子）。两轨/三轨结果**不一致**时：不得简单取其一，**必须定位错源（哪一轨错、错在哪一步）**，按 **BLOCKER 级**处置、定位并修复前不得落盘。适用范围仅限同时满足"决策关键"（进不可逆/强约束）+"单工具自证有 bug 风险"（一行公式笔误即可量级偏差、肉眼难辨）的数字；普通中间过程量、已有 L1 实测背书量、低风险初选量**不强制**（避免成本失控）。缘起 LESSON-012：M1 WNG 草稿 bug（多除 N，约 12dB 量级偏差）在落盘前被三轨交叉核当场抓出并精确定位为"numpy 错、MATLAB/CTO 对"，若单轨开发将直接落盘。

## 4B. 三道关：workflow 产出验证闸（v1.8 新增，CTO 拍板 2026-06-04）

> **类型**：强制流程制度（治理 agent-workflow 输出质量，非数字定级）。与铁律一–九（管数字 provenance）
>   正交：铁律管「数字身份证」，本条管「产出经过几道独立检验」。

**核心原则**：**workflow/多 agent 的每一轮输出都可能引入新错——包括「修正错误的那一轮」。**
任何 workflow 产出（设计、诊断、补丁、数字、报告，**含修正稿/二次修正稿**）在被采信或下游引用前，
**必须依次通过三道关，缺一不可**：

1. **自动 verify（初筛，NOT 门）**：workflow 内部的 verifier/自检/自审。仅作初筛过滤，**不构成门禁**——
   它与产出者同上下文，可被同一盲点污染（甚至撤销自己已做对的纠正，见缘起 R8）。
2. **独立 critic 门（THE GATE）**：全新上下文的独立 critic teammate，出具带 `reviewer: critic @
   <exact model ID> / <date>` 的 verdict。这是唯一放行门。in-context 调 critic skill 自审 **不算**
   （team_config Commit-discipline / Fallback 条款一致）。
3. **CTO 常识合理性审（FINAL）**：CTO 对数量级/方向/口径做常识性 sanity check（如「2.9/1000 该是 ~340x
   不是 49x」——本条缘起之一）。前两道关均可能放过量级/口径错，CTO 审是最后兜底。

**红线**：**不得假设「修过即对」。** 修正稿与原稿同等过三道关；修正动作本身是新一轮 workflow 输出，
同样可能引入/遗漏错（缘起 R9：修正稿自带两处偏乐观新错）。critic 评审报告须显式列三道关执行状态。

**缘起**：R8（2026-06-04，synthesizer 撤销其内部 verifier 已纠正的 21.7 MCPS 错下界——「自动 verify」被
同上下文推翻）+ R9（STEER-2 「49x」修正稿自带 2.55x 低端算错应 2.31x、3 cyc/MAC 无板证下界——修正稿新错）。
两案证明：单靠 workflow 自检不足；独立 critic + CTO 常识审是命门。

## 8. 完整示例 —— PF-1（算力裕量）：旧错误写法 vs 新正确写法

### ❌ 旧写法（违反制度，导致 PF-1）
> decisions_log DEC-S1-004：
> "单核算力 1GHz 满足 4 子带 DAS 波束形成（树形结构 **56.4 MMAC/s，裕量 27×**，可行）"
> 状态：✅ LOCKED（芯片选型不可逆采购）

**问题诊断（按本制度）**：
- ✗ 违反 **C1**：56.4/27× 无来源等级标注。
- ✗ 违反 **C3 + 铁律一**：27× 实为 **L3 解析估算**（树形结构当时从未落 C），却作了**不可逆决策（芯片 LOCKED + 采购）的唯一依据**。
- ✗ 违反 **C4 + 铁律二**：L3 支撑芯片选型却未挂"待 EZKIT L1 验证"标记。
