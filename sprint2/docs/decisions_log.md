## Sprint 1 决策（已完结）

### DEC-S1-001：技术路线选定
- **决策内容**：采用相控阵线阵（DAS + 子带波束形成）  
- **决策依据**：竞品拆机分析证实主流路线为 ADI 系芯片驱动的线阵相控阵；DAS 算法成熟、资源消耗低；子带分频可满足多频段独立波束控制  
- **决策方**：CTO（总工）  
- **决策时间**：Sprint 1  
- **状态**：✅ LOCKED

---

### DEC-S1-002：指向性频段需求
- **决策内容**：仅 ≥1kHz 要求强指向性；250～500Hz 不作指向性要求  
- **决策依据**：低频波长长（500Hz 对应 0.68m），16 元 30mm 阵在此频段物理上无法形成窄波束；CTO 明确用户需求集中在语音清晰度频段（1kHz 以上）  
- **决策方**：CTO（总工）  
- **决策时间**：Sprint 1  
- **状态**：✅ LOCKED  
- **影响**：子带设计从 4 个子带（500-1k/1k-2k/2k-4k/4k-8k）执行时，低频子带 500-1k Hz 可降低波束形成精度要求，重点保障 1k-8k Hz 性能

### DEC-S1-003：Gate 1 基线阵列验证通过
- **决策内容**：16 元 / d=30mm 基线设计通过 Gate 1 可行性验证；SHARC 算力可行性确认

- **决策方**：Project Manager Agent + Critic Agent，经 CTO 认可  
- **决策时间**：Sprint 1 末  
- **状态**：✅ LOCKED（作为 Sprint 2 比较基准）

### DEC-S1-004：DSP 芯片型号锁定
- **决策内容**：选定 ADSP-21569（SHARC+ 单核 1GHz，640KB L1 + 1024KB L2，400-ball CSP BGA 17×17mm）  
- **决策依据**：  
  1. 不需要 ARM 核（无操作系统/网络/显示屏需求），ADSP-21569 纯 SHARC+ 架构更节省成本和 BOM 复杂度  
  2. 单核算力 1GHz 满足 4 子带 DAS 波束形成（树形结构 56.4 MMAC/s，裕量 27×）  
  3. SRAM 1664KB 总量，实际占用仅 27.3KB（1.6%），余量充足  
  4. 开发工具：CrossCore Embedded Studio（CCES）免费，EV-21569-EZKIT 开发板可购  
  5. 中端定位，单价 ¥120-220（1k 量级估算），符合 BOM 成本目标  
- **决策方**：CTO（总工）  
- **决策时间**：Sprint 1  
- **状态**：✅ LOCKED（不可逆，后续方案设计均以此为基准）

### DEC-S2-002：DSP 实现方案确定（树形 FIR 结构）
- **决策内容**：采用 dyadic 树形半带 FIR 分频结构（而非全速率直接 FIR），48kHz 采样率，4 子带（500-1k/1k-2k/2k-4k/4k-8k）  
- **决策依据**：  
  - 全速率方案：1480 MMAC/s，超出 ADSP-21569 峰值算力，不可行  
  - dyadic 树形方案：56.4 MMAC/s，裕量 27×，可行  
  - 核心波束形成：6.48 MMAC/s（16ch 情况）  
  - SRAM 占用：27.3KB（总量 1664KB 的 1.6%）  
  - 真实 Kaiser FIR 阻带：58-73dB（满足子带隔离需求）  
- **残留风险**：树形结构 C 代码尚未实现（Sprint 2 仅建立全速率参考实现），Sprint 3 需完成 CCES 移植验证  
- **决策方**：DSP 算法 Agent，经 Critic Agent 审核  
- **决策时间**：Sprint 2  
- **状态**：✅ LOCKED（算法路线锁定，实现待 Sprint 3 完成）

### DEC-S5-EQ-O1-01：item-3 = O1（LEAN master-bus EQ + 保护限幅 REQUIRED）
- **裁定（substance 逐字）**：EQ 取 O1。
- **O1 规格（EQ_PRD_DECISION_MATERIAL.md:69 + §2.2 结构规则）**：
  - **整形 EQ**：2–3 biquad，**master-bus（fan-out 前一次处理）**。基础 = broadside 同信号 + 整形 EQ/per-ch 延迟
    均 LTI 可交换（master-bus EQ→fan-out→per-ch 延迟 ≡ per-ch EQ→延迟，频谱整形等价），broadside 与聚焦下
    均成立、约 1/8 成本（EQ_PRD §2.2:50/52）。
  - **保护限幅器 = REQUIRED 入 PRD**（三场景皆需；3W 实测低功率喇叭[L1]+D 类功放，削波灌音圈风险，EQ_PRD §2:44）。
    **摆位 = per-channel（D-C 权重 fan-out 之后）**：各通道 D-C 权重不同（中心 w=1.0、次边 w≈0.50，~6dB 跨度），
    单一 master 限幅器保护不了最高权重通道（SN-5）。**阈值按权重缩放 T_k=T·w[k]**（D14）以保锥度——
    共阈值 per-channel 限幅会先削高权重通道、把锥度推向均匀加权 → 旁瓣抬升、-20dB SLL 劣化（EQ_PRD §2.2:54）。
    限幅器标量/近零 MAC，8 个 per-channel 可忽略。
  - **band 数 = 终调待 R3**：是否真需 EQ、几频段取决于实测正轴向响应（R3 消声室，待 T/S 2–4 周，EQ_PRD:45）。

- **成本 [L4]**：29–60 MCPS（=20–25 MAC/samp×48000×{30,50}cyc/MAC；28.8@2bq/30cyc → 60.0@3bq/50cyc）。

  非 O1 值——勿混（EQ_PRD:69 critic 修正记录）。**
- **HW-1 条件性（O1）= 有价值但非承重**（EQ_PRD:81）：offload 2–3 biquad 后 core 仅余限幅+编排；
