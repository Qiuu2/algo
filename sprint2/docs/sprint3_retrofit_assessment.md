## 1. 改造方案技术风险评估

### 1.1 ⚠️ 核心隐患：几何不匹配（裁决项）

**事实对比**：

| 维度 | 我方算法基线 | 竞品实物（反推） | 是否匹配 |
|------|------------|----------------|---------|
| 阵元数 N | 16 | ≈20（候选含 24/32） | ❌ 不匹配 |
| 间距 d | 30mm | 35mm | ❌ 不匹配 |
| 阵列总长 L | 450mm | 665mm | ❌ 不匹配 |
| 加权 | Dolph −20dB | Dolph −20dB（拟合） | ✅ 框架一致 |
| 通道数 | 16ch | ≈20ch | ❌ 不匹配（且 16<20，路数不足）|

> ⚠️ **反推本身是估计值**：N=20/d=35 是声学 Agent 以远场点声源模型对 6 个实测角度拟合得到，4kHz 置信度仅中等，且竞品可能含未知 DSP 均衡/相位补偿。真实几何**必须拆机物理点数+卡尺实测**才能定论。

#### (a) 这是否破坏"验证我方设计"的前提？

**是，部分破坏。** 我方算法的核心产出是"针对 16 元 / 30mm 几何计算出的每通道加权 w_n 与子带 FIR 延迟系数"。这套系数与**具体的阵元位置坐标强绑定**：波束形成本质是 `AF(θ)=Σ w_n·exp(j·k·n·d·sinθ)`，一旦 d 从 30mm 变 35mm、n 从 16 变 20，原系数产生的波束指向/波宽/旁瓣全部失真。

- 直接把我方 16ch 系数灌进竞品 20 元阵列：**前 16 个驱动单元被驱动、后 4 个无信号**（或映射错乱），且每个单元的相对相位按 30mm 算、实际却是 35mm 间隔 → 波束**不会**形成预期的 −6dB≤30°@2kHz，而是散焦、旁瓣抬高、指向角偏移。**这不是"验证我方设计",而是"验证一个错误耦合的系统"。**

#### (b) 实际能验证什么 / 不能验证什么？

| ✅ 能验证（若做几何重定向后） | ❌ 不能验证 |
|---------------------------|-----------|
| 我方**算法框架/代码链路**能在 ADSP-21569 真实硬件上跑通（FAS 树形结构、子带分解、定点化、DMA ping-pong） | 我方 **16元/30mm 几何方案本身**的声学性能（竞品是 20元/35mm，几何不是我方的）|
| **21569 平台**的实测算力裕量（MCPS）、端到端延迟（R1/R2 闭环目标） | 我方**自研 PCB**的电气/EMC/热表现（用的是 EZKIT，不是自研板）|
| 算法驱动**真实喇叭单元 + 真实箱体**的**声-电耦合效果**（互耦、障板衍射、单元一致性等理想化假设的破除）| 我方选型的**扬声器单元**（用的是竞品的喇叭，不是我方选型单元）|
| 波束形成**在实物阵列上能否真的成形**（定性验证算法有效性）| 我方**绝对 SPL** 对标（竞品功放+竞品喇叭的链路，非我方功率预算）|

#### (c) 是否必须先拆机确认几何、再把权重/延迟重定向到竞品实测几何？

**是，这是本方案有意义的前提条件，不可省。** 正确做法：

1. **拆机**：物理点数确认竞品真实阵元数 N、卡尺实测间距 d、确认是否分组/子阵、确认是否有内置无源分频或 DSP。记录在案（法律约束）。
2. **几何重定向**：以竞品**实测**的 N、d 重新运行我方算法权重/延迟生成（声学 Agent + DSP Agent 协作），生成"针对竞品几何"的系数集。
3. **通道数适配**：若竞品 N=20 而 EZKIT/转接板只能出 16ch，则**要么扩到 ≥20ch 输出**（见第 2 节），**要么只驱动竞品阵列的连续 16 个单元、其余短接/空置**（会改变有效孔径，需声学重算预测）。

#### (d) 诚实裁决

> **本方案验证的是："算法框架 + ADSP-21569 平台 + 真实喇叭/箱体声耦合" 的可行性。**
> **本方案 *不* 验证 "我方 16元/30mm 几何方案" —— 因为物理阵列是竞品的 20元/35mm，不是我方的。**

换言之，这是一次**"算法引擎 + 真实声学负载"的台架联调**，不是**"我方产品几何"的验证**。要让它"验证我方设计"，唯一途径是**把算法重定向到竞品实测几何**——但那时验证的也是"我方算法对 20/35 几何的适配能力"，仍非"我方 16/30 几何"。**我方 16/30 几何的真实验证，只能靠我方自研阵列（原 Sprint 3 计划的 S3-G5 消声室实测路线）。**

### 1.2 其它技术风险

| 风险 | 等级 | 说明 | 处置 |
|------|------|------|------|
| **竞品功放输入接口未知** | 🔴 P0 | 模拟线路输入？数字 I2S/TDM？还是功放与喇叭一体不可分离？决定整个转接方案。**必须拆机逆向**。 | 拆机优先项；逆向前不可设计转接板 |

## 2. 转接板关键设计要点

### 2.1 ⚠️ EZKIT 真实音频 I/O 能力核实（关键裁决）

**已核实**（ADI 官方资料，见文末来源）：

> **EV-21569-EZKIT 板载音频 I/O：**
> - DAC：**ADAU1962A，12 通道**，24bit/192kHz，引出 **12 路 RCA 模拟输出**（或 8 出 / 4 入配置）。
> - ADC：**ADAU1979，4 通道**输入。
> - 板载 Codec 通过 **SPORT（TDM）**与 21569 连接。

**结论**：
- 板载模拟输出**最多 12 路** < 我方 16ch < 竞品 ≈20ch。**16/20 路模拟输出，板载绝对不够。**
- 早先 BOM v0 中"ADAU1962A=8ch"的记述应更正为**12ch**（实物为 12ch DAC）。即便按 12ch，仍不足 16/20。

> ✅ **裁决：转接板必须自带多通道 DAC（如 ADAU1966A，16ch），由 EZKIT 的 SPORT/TDM 馈入**，而非依赖 EZKIT 板载 12ch 输出。
> - 16ch 验证：转接板挂 **1× ADAU1966A（16ch）**，EZKIT SPORT0 出 1 路 TDM16 → 转接板 DAC。
> - 20ch（重定向到竞品几何）验证：需 **2× ADAU1962A(12ch)** 或 **ADAU1966A + ADAU1962A**，用 2 路 TDM（SPORT0+1），取前 20 路。
> - **替代捷径**（仅当通道数 ≤12 时）：若决定先只验证 12 个连续单元（缩减孔径、声学重算预测），可暂用 EZKIT 板载 12ch RCA 直驱，省掉转接板 DAC——但波束会因孔径缩短而展宽，仅作定性联调，不推荐作为主路径。

### 2.2 两种接入情形分别给方案

#### 情形①：竞品功放是【模拟线路输入】

```
EZKIT (21569) ──SPORT/TDM──► 转接板 DAC (ADAU1966A 16ch / +ADAU1962A 凑20ch)
                                     │ 16/20 路单端或差分模拟输出
                                     ▼
                        [电平/阻抗匹配网络 + 限幅保护]
                          - 串联衰减（按竞品功放输入灵敏度实测定）
                          - 阻抗匹配（DAC 低阻输出 → 竞品功放高阻输入）
                          - RC 限幅 + TVS/钳位保护（防过驱动烧功放）
                                     │
                                     ▼
                        竞品功放各通道【模拟输入端】（拆机定位）
                                     │
                                     ▼
                        竞品功放 → 竞品喇叭阵列（保留原样）
```
转接板必备：**16ch DAC（自带）+ 输出滤波（重建 LPF）+ 每通道电平/阻抗匹配 + 限幅保护 + 逐通道测试点**。

#### 情形②：竞品功放是【数字 I2S/TDM 输入的 D 类】

```
EZKIT (21569) ──SPORT/TDM──► [转接板：电平转换/缓冲/时钟适配] ──► 竞品功放数字输入
                                     │
                              - 若竞品功放为 3.3V I2S：电平直配或缓冲
                              - 若 TDM 协议/槽数/采样率不同：转接板需做格式适配（或 21569 侧软件配置匹配）
                              - MCLK/BCLK/FSYNC 来源：用 EZKIT 主时钟 还是 竞品功放时钟？须定主从
```
此情形**可省掉转接板 DAC**（直接数字馈入），但要求竞品功放数字接口协议**完全逆向清楚**（槽宽、槽数、采样率、电平、时钟主从）。**这是最优捷径——若竞品确为数字输入 D 类功放，方案复杂度与风险大幅下降。**

> ⚠️ 情形①还是②，**完全取决于拆机逆向结果**。逆向前两套方案都要预留，逆向后才能定型。

### 2.3 其它转接板设计要点

| 要点 | 设计要求 |
|------|---------|
| **电源隔离** | EZKIT 用自带 12V/USB 供电；竞品功放保留原供电。两者**信号地策略**：优先单点共地（消除地环路嗡声）；若两套电源浮地差异大，转接板模拟信号路径用**音频隔离变压器**或**数字隔离器（情形②的 I2S 用 ADuM 系列）**做地隔离。**禁止让 EZKIT 给竞品功放供电**（功率不够且地耦合）。 |
| **上电时序** | 先上 EZKIT + 转接板 DAC（建立稳定静音输出）→ 再上竞品功放。下电反序。防止上电瞬态脉冲冲击竞品功放输入烧机。转接板加**上电静音（mute）电路**：DAC 未稳定前钳位输出到 0。 |
| **机械固定** | 转接板与 EZKIT 通过标准排针/排线连接（不改 EZKIT 板）；与竞品功放的连接用**飞线+测试夹/临时接插件**，避免在竞品 PCB 上大量焊接（降低不可逆损伤、便于复原记录）。转接板独立固定，不悬空（防短路）。 |
| **连接器适配** | 输入侧：匹配 EZKIT SPORT/扩展接口排针定义（查 EZKIT 原理图）。输出侧：按拆机逆向的竞品功放接口现状定（排针/排线/焊盘）。 |
| **通道映射可调试性** | 🔑 **每通道必须可独立验证**：转接板每路 DAC 输出留**独立测试点**；支持单通道注入已知测试信号（如仅第 n 路出 1kHz 正弦），逐路确认"DAC 通道号 → 转接板线序 → 竞品功放通道 → 物理喇叭位置"映射正确。建议转接板加**通道指示 LED 或拨码选通**，便于排查映射错配（1.2 节 P0 风险）。 |

## 3. EZKIT ↔ 竞品功放 电平/时序兼容性分析

### 3.1 情形①：模拟接口兼容性

| 参数 | EZKIT/转接板 DAC 侧 | 竞品功放输入侧 | 兼容性判断 | 是否需先拆机实测 |
|------|-------------------|---------------|-----------|----------------|
| 电压摆幅 | ADAU1966A 差分满幅 ≈ ±1.4V（约 2.8Vpp 差分 / 1Vrms 量级，单端约半值） | **未知**（消费功放灵敏度常见 0.3~2Vrms 满输入） | 须匹配，可能需衰减或增益 | ✅ **必须实测竞品满量程输入灵敏度** |
| 阻抗 | DAC 输出阻抗低（<100Ω，需缓冲后驱动） | **未知**（功放输入阻抗常见 10k~47kΩ） | 高阻输入易匹配，但需确认 | ✅ **必须实测竞品输入阻抗** |
| 直流偏置 | DAC 输出可能含 DC bias，需 AC 耦合 | 竞品输入是否 AC 耦合未知 | 转接板加隔直电容保险 | ✅ 拆机确认耦合方式 |
| 过驱动保护 | 转接板加限幅 | 过压烧输入级风险 | 必须限幅 | ✅ 由灵敏度推限幅阈值 |

### 3.2 情形②：数字接口（I2S/TDM）兼容性

| 参数 | EZKIT (21569) 侧 | 竞品功放数字输入侧 | 兼容性判断 | 是否需先拆机实测 |
|------|-----------------|------------------|-----------|----------------|
| 逻辑电平 | 21569 SPORT 通常 **3.3V** CMOS | **未知**（可能 3.3V 或 1.8V） | 若 1.8V 需电平转换 | ✅ **必须实测/查竞品功放芯片手册** |
| BCLK | 我方 16ch：24.576MHz（48kHz×32bit×16slot）；20ch 更高 | 竞品功放 D 类支持的 BCLK 上限未知 | 须确认竞品支持的最高 BCLK 与槽数 | ✅ 必须确认 |
| FSYNC/LRCLK | 48kHz | 竞品采样率未知（可能 48k/44.1k/96k） | **采样率必须匹配** | ✅ 必须确认 |
| MCLK | EZKIT 可提供 12.288/24.576MHz | 竞品功放 MCLK 需求/主从未知 | 须定时钟主从（建议 EZKIT 做主） | ✅ 必须确认 |
| TDM 槽数/槽宽 | 我方 16 槽 ×32bit | 竞品 D 类支持的槽配置未知 | 须匹配或软件可配 | ✅ 必须确认 |
| 上电/静音时序 | 21569 软件可控 mute | D 类功放上电 pop 抑制时序 | 先建链路再解 mute | ✅ 确认竞品 mute 引脚/时序 |

### 3.3 必须"先拆机实测"才能定的清单（汇总）

> 以下项目**逆向前无法纸面确定**，是本方案能否落地的**决定性未知量**：
1. 竞品功放输入是**模拟**还是**数字**（决定走情形①还是②）。
2. 竞品真实阵元数 N、间距 d、是否分组（决定几何重定向）。
3. （模拟）竞品功放输入灵敏度、阻抗、耦合方式（决定匹配/限幅，防烧机）。
4. （数字）竞品功放数字接口协议：电平、BCLK、采样率、TDM 槽配置、MCLK 主从、mute 时序。
5. 竞品主控是否含 DSP/EQ/分频（决定旁路后信号链是否失真）。
6. 竞品供电与地结构（决定隔离/共地策略）。

| 扬声器单元选型/询样/送样/比价 | 3–4 周（货期主导） | S3-G4，用竞品喇叭替代 |
| 阵列承载结构件/箱体设计打样 | 2–3 周 | 用竞品箱体替代 |
| **小计（省）** | **约 9–13 周 ≈ 2–3 个月** | （部分可并行，实际净省小于总和）|

### 4.2 新增的工作（快速验证法引入）

| 新增项 | 耗时（估） | 风险 |
|--------|----------|------|
| 竞品拆机 + 功放接口逆向 | 1–2 周 | 接口未知，可能逆向困难/失败 |
| 转接板设计（含 16ch DAC、匹配、保护、可调试） | 1.5–2 周 | 比"无 DAC 转接板"复杂得多 |
| 转接板 PCB 打样 + 焊接 | 1.5–2 周 | 仍需打一块 PCB（只是比主板小）|
| EZKIT↔转接板↔竞品功放 接口调试 | 1–2 周 | 电平/时序/映射逐项调通 |
| 几何重定向（算法权重/延迟重算 + 验证） | 0.5–1 周 | 声学+DSP 协作 |
| 改装风险缓冲（烧功放/损样机返工） | 0.5–1 周 | 不可控 |
| **小计（增）** | **约 6–10 周 ≈ 1.5–2.5 个月** | |

### 4.3 净节省现实估计

```
净节省 ≈ 省（9–13 周）− 增（6–10 周）= 约 +0 ~ +5 周（乐观），中值 ≈ 2–3 周
```

> **诚实结论：声称"省 3 个月"过于乐观。现实净节省约 0.5–1 个月（2–4 周），且伴随显著的逆向/改装不确定性风险。** 之所以省得不多，是因为：
> - 转接板仍需自己设计 + 自带 16ch DAC + 打样 + 调试，**并没有真正省掉"做一块 PCB"**，只是把"主板"换成了"转接板"（复杂度未必更低，因为多了逆向适配）。
> - 拆机逆向是**新增的、原计划没有的**高不确定性工作，可能成为新瓶颈。
> - 几何不匹配导致**验证结论本身打折扣**（见第 1 节）——省了时间却没验证到"我方几何"。

### 4.4 这是"加速验证里程碑"还是"替代产品开发"？

> **这是"加速验证里程碑"（提前拿到 算法+平台+真实声耦合 的联调结论），*不是* 替代产品开发。**
> 即使快速验证成功，**第四阶段（正式产品）仍必须走原自研流程**：自研 PCB、自选扬声器单元、自研箱体结构、我方 16/30 几何的消声室实测（原 S3-G5）。快速验证法**不能替代**这些工作，只能**提前降低算法/平台层面的风险**，避免在自研 PCB 投入后才发现算法-真实声学耦合的问题。

## 5. 总体建议

**评级：🟡 有条件推荐（作为低成本探针，不作为主验证路径）**

**推荐理由（支持做）**：
- 能用很低成本**提前**拿到"我方算法引擎能否在 21569 上跑通 + 能否驱动真实喇叭成形波束"的定性结论，降低自研 PCB 的投入风险。
- 与原 Sprint 3 的 R1（EZKIT 实测算力）/R2（延迟）目标**天然重叠**——这两项本来就要在 EZKIT 上做，借此顺带挂真实喇叭联调。

**附加条件（必须满足才做）**：
1. **先拆机逆向，后定方案**：拆机确认①模拟/数字接口 ②真实 N/d ③功放电平阻抗。逆向不清楚则**不启动转接板设计**。
2. **几何必须重定向**：算法权重/延迟按竞品**实测**几何重算，否则验证无意义（验的是错系统）。
3. **明确验证边界**：对内对外都讲清——这验证的是"算法框架+21569+真实声耦合"，**不是"我方 16/30 几何"**。几何验证仍走自研阵列消声室路线。
4. **保护优先**：转接板必带限幅/隔离/上电静音/逐通道可调试，防烧竞品功放、防地环路、防映射错配。
5. **保留对照样机**：至少留 1 台未拆竞品作对照，仅改装 1 台（法律+工程双重考虑）。
6. **不夸大时间收益**：以"净省约 2–4 周 + 提前降风险"为预期，不以"省 3 个月"为决策依据。

**不建议把它当作"替代自研"或"我方几何验证"的路径** —— 第四阶段自研流程不可省。
