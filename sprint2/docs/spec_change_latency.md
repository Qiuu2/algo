# 规格变更说明（草稿）— 系统端到端延迟

**文档编号**：DOC-SPEC-CHG-001
**版本**：v0.1（草稿，待市场/客户对齐后正式生效）
**日期**：2026-05-26
**关联决策**：DEC-S2-012
**状态**：⏳ 草稿 — CTO 将据此与市场/客户对齐，对齐通过后正式生效

> 用途说明：本草稿供 CTO 拿去与市场部 / 目标客户对齐。对齐确认"播放延迟不构成使用障碍"后，正式纳入 PRD 并关闭遗留风险 R2。

---

## 1. 变更概要

| 项目 | 原规格 | 新规格 |
|------|--------|--------|
| 系统端到端延迟（节目源输入 → 扬声器声输出） | **< 5 ms** | **< 30 ms** |
| 解析/仿真当前值（非硬件实测） | — | **12.53 ms** |
| 变更性质 | 放宽（relax） | 仍有 ~17ms 设计余量 |

---

## 2. 原规格的来源与问题

- 原 <5ms 指标沿用了**通信 / 会议 / 监听**类音频系统的惯例值——这类系统存在**对话回路、唇同步、艺人返听**等硬实时约束，故要求极低延迟。
- 本品（定向音柱）的实际使用场景为**单向播放**：博物馆讲解、车站/机场广播、商场分区广播。**不存在**对话回路、不存在唇同步约束、不存在返听需求。
- 因此 <5ms 对本品而言是一条**过严且无场景依据**的继承指标。

## 3. 工程核算（变更依据，scipy 解析/仿真，**非硬件实测**）

DSP 端到端延迟分解（scipy 解析计算 `latency_calc.py`，**非硬件实测，待 EZKIT 上板确认**）：

| 环节 | 延迟 | 说明 |
|------|------|------|
| dyadic 树形滤波器组（分析 4.52 + 合成 4.52） | 9.04 ms | 多速率重建组延迟，低速率级贡献最大 |
| DMA ping-pong（输入 + 输出，2 帧） | 2.67 ms | 64 样点/帧固定开销 |
| ADAU1966A DAC 群延迟 | 0.75 ms | 输入为数字节目源、无 ADC；仅输出 DAC |
| 分数延迟 FIR | 0.07 ms | 可忽略 |
| **合计** | **12.53 ms** | |

**为何不能简单压到 <5ms**：
- DMA + DAC 的**固定开销已 3.49ms**，留给滤波器组的预算只剩 1.51ms。
- 全速率 FIR 方案虽组延迟仅 4.54ms，但算力 1480 MMAC/s，**超 ADSP-21569 包络，不可行**。
- 降阶 437→257 抽头可把 FIR 组延迟降到 2.67ms，但：① 阻带衰减下降（SB1 关键指向子带 70→67dB，裕量转紧）；② 加上固定开销**仍进不了 5ms**。
- 结论：在保留高质量滤波器组（阻带 58–73dB）+ 完美重建的前提下，**5ms 物理上不可达**；强行压低只会牺牲指向性质量。

## 4. 新规格的合理性（人因依据）

- 人耳对**单向播放**的绝对延迟在 **< 20–30ms** 区间**无感知**（无参照对比时）。
- 12.53ms 远在该无感区间内，且对"声音是否准时播放"无任何主观影响。
- 新规格 <30ms 预留了约 17ms 余量，可吸纳后续 1kHz 优化（DEC-S2-013）等附加处理带来的延迟增量。

## 5. 影响分析

| 维度 | 影响 | 说明 |
|------|------|------|
| 音质 / 指向性 | ✅ 正面 | 得以保留高质量线性相位滤波器组与完美重建，指向性不打折 |
| 算力 | ✅ 无影响 | 维持 dyadic 树形 56.4 MMAC/s（27× 裕量） |
| 硬件 / BOM | ✅ 无影响 | 零硬件变更 |
| 遗留风险 R2 | ✅ 关闭 | 不再需要降延迟改造 |
| 后续 1kHz 优化 | ✅ 兼容 | DEC-S2-013 叠加延迟仍在 30ms 预算内（精确值 Sprint 3 合并核算） |
| 客户感知 | ⚠️ 待对齐 | 需市场/客户确认其应用无低延迟硬需求（见第 6 节） |

## 6. 待对齐事项（CTO 执行）

正式生效前需向市场/客户确认：

1. 目标客户的应用是否**全部为单向播放**？是否存在任何**实时互动 / 同步**需求（如与视频严格同步、多音柱级联需相位对齐的扩声场景）？
2. 多台音柱**级联组网**时，各台延迟一致性要求是什么？（同批次同算法延迟一致，但需确认）
3. 是否有客户合同/招标书中已写入 <5ms 或其他延迟硬指标？

> 若上述均无冲突 → 新规格 <30ms 正式生效，更新 PRD，关闭 R2。
> 若存在低延迟硬需求客户 → 回到 DEC-S2-012 复议（可能需为特定客户提供低延迟/低质量模式）。
