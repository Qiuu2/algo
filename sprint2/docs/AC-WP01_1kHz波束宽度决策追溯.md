`29.28°` 不是单一来源，而是 **4 种相互独立的方法全部收敛到同一答案**：

| # | 方法 | 执行方 | 结果 |
|---|------|--------|------|

## 3. 错误根因：14.63° 是怎么来的、为什么错

**结论先行：numpy 全程正确；bug 仅存在于 MATLAB 脚本 `sim_d55_beamwidth.m`，且已修复。**

### 14.63° 的本质
`14.63°` = 单边半角（−6dB 交点在 ±14.64°），即真实全角 `29.28°` 的**一半**。它被 AC-WP01 初版误当成了"波束宽度"。

### 出错位置
- 文件：`~/matlab-agent/sim_d55_beamwidth.m`
- 函数：`local_beamwidth`（v1 自写的取宽逻辑）

### 出错机理（已用 MATLAB 诊断坐实，非推测）
1. 角度网格为 `ang = −180:0.1:180`；
2. `[peak, ipk] = max(resp)` 本想取 0° 主瓣峰，但 **broadside + isotropic 阵元** 下，±180° 端射瓣与 0° 主瓣**等高**；
3. `max()` 返回**首个**最大索引 → `ipk` 落在 **−180°**（端射瓣），而非 0°；
4. 向左走的索引立刻撞上数组左边界（−180°），向右走到 −165.37°；
5. 测出的"宽度" = −165.37 − (−180) = **14.63°** —— 这是 **−180° 端射瓣的单边半宽**，因瓣型与真主瓣全等，恰好 = 真全角 29.28° 的一半。

归类：**峰值定位错误（max 选中端点瓣 + 左边界截断）**，而非"只算单边漏 ×2"或"把交点角当宽度"。

### 为什么 BW_cos 当时侥幸正确
cosine 阵元方向性压制了 ±180° 端射瓣，主瓣峰唯一落在 0°，`max` 选对了，故 cos 列未受影响。

### 为什么 numpy 没有这个问题
numpy 各 `beamwidth_6db` 在 **0..90° 半空间网格**上以 `db[0]`（θ=0°）为峰，结构上不存在 ±180° 端点瓣陷阱；`return 2 * t_6dB`（半角 × 2 = 全角）是教科书正确做法。**numpy 无因子-2 bug，所谓"2 倍偏差"纯属 MATLAB v1 半角 bug 的镜像错觉。**

### 修复
`sim_d55_beamwidth.m` 已改用 MATLAB 内置 `beamwidth()` 的第一返回值（全角），丢弃第二返回值 `ang`（半角交点）；BW_iso 与 BW_cos 共用同一全角口径。修复后 5 频点与 numpy 逐点吻合。

## 4. 一秒判据（反转决策前先用此核验）

> **−6dB 全角波束宽度 必然 > −3dB 全角波束宽度。**

主瓣从峰值单调下降，先过 −3dB 点、再过 −6dB 点，所以 −6dB 等高线一定比 −3dB 更宽。

- 本例：−3dB 全角 = **21.24°**，−6dB 全角 = **29.28°**，29.28 > 21.24 ✓
- 当初若有人用这一句话核验：`14.63° < 21.24°（−3dB 全角）`，**−6dB 波束宽度比 −3dB 还窄，物理上不可能** → 立刻能判定 14.63° 是错的（半角或 bug）。

**项目铁律（写入规范）**：
1. **BW = 全角 −6dB 波束宽度 = 2 × 单边半角**；任何单边半角必须显式标注 `±` / `half`，禁止与全角混用。
2. MATLAB 侧优先用 `beamwidth()` 的**第一返回值（全角）**，**禁用第二返回值 `ang`（半角交点）**。
3. 自写取宽必须：① 断言峰值在 broadside(0°)；② 用 −3dB 全角做下界 sanity check。
4. **任何会推翻既有决策的数字，转报 CTO / 据此行动前，先用上面的一秒判据或独立复算核验。**

## 6. 竞品基准失效说明

**竞品 1kHz 波束宽度目前没有任何可信值**，两个流传的数字都不能用：

| 数字 | 来源 | 为什么不可信 |
|------|------|------------|
| **16°** | 竞品实测，但用仅 4 个角度（0/30/60/90°）线性插值 | 角度采样太稀（30° 步长）必然**低估**主瓣宽度；已被 `sweep_report.md §7.1 / F-AC-01` 撤回 |
| **36.6°** | 反推模型隐含值（DEC-S2-001） | 按**错误几何 N≈20/d=35** 反推；拆机真值是 **N=16/d=55**（DEC-S3-003），几何前提已废止。该值本身是 N=20/d=35 配置的合法全角（MATLAB 复算 36.64° 吻合），但**不代表真实竞品** |

- "14.63° ≈ 竞品 16°"的所谓"吻合"是**双重错误的巧合**：AC-WP01 半角(14.63°) ≈ 稀疏插值低估值(16°)，两个错的数凑巧接近，不构成任何佐证。
- **结论：真实竞品 1kHz 波束宽度只能靠消声室实测获得。** 在此之前，不要用 16° 或 36.6° 作为我方设计/超指向决策的对标依据。
