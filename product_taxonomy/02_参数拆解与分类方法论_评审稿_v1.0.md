# 产品参数拆解与分类方法论（评审稿 v1.0）

> 目的：说清楚三件事——**怎么分类、怎么定义参数字典、怎么逐行拆解参数**，供决策人评审可行性。
> 评审通过后，此文档即为全线产品参数整理的执行标准。
> 配套文件：`00_分类树草案_v0.1.md`（分类树全文）、`01_参数字典样板_v0.1.md`（5 个类目的字典样板）。
> 日期：2026-07-03

---

## 一、总体思路与最终产出物

**核心原则：不建全品类统一大表。** 分类树负责"这是什么产品"，参数模板挂在每个末级分类上，同类产品参数列完全一致、可直接排序比对；跨类产品只共享少量公共列。这是电商类目体系和工业品分类标准（ETIM/eCl@ss）的通行做法。

最终交付四样东西：

| # | 产出物 | 形态 | 作用 |
|---|---|---|---|
| 1 | 分类索引表 | 一张总表 | 全部产品 → 大类/子类/孙类/分类编码 的映射 |
| 2 | 参数字典 | 每个末级分类一张定义表 | 规定该类有哪些拆解列、类型、单位、允许值、别名 |
| 3 | 产品宽表 | 每个末级分类一张表（或一个 sheet） | 原始参数原文 + 拆解列 + 解析状态，AI 直接可用 |
| 4 | 复核清单 | 一张汇总表 | 所有标了`推断/冲突/缺失`的条目，供人工确认 |

宽表的列布局（即"原参数后面追加十几列"的落地形式）：

```
[原始参数列(厂家原文逐字保留)] | [公共列] | [该类专属拆解列] | [解析状态] | [备注说明]
```

## 二、怎么分类

### 2.1 三级结构与判定标准

- **大类**：按业务线/系统划分（公共广播、专业音响、会议、录播教育、信号中控、LED显示、安防监控、舞台灯光、配件）。判定标准：属于哪套系统方案。
- **子类**：按设备职能划分（如公共广播下：广播功放/控制前端/IP网络广播/扬声器/消防应急）。判定标准：在系统里干什么活。
- **孙类（末级）**：按参数结构划分。判定标准是唯一一条硬规矩——

> **同一末级分类内，所有产品必须能用同一套参数列描述，且这些列对每个产品基本都有值。**
> 若一个孙类下出现两群产品各自有一批对方没有的参数（例如"网络终端"里带功放的和不带功放的），就拆成两个孙类。反过来，两个孙类如果参数列几乎重合，就合并。

这条规矩决定了分类不是拍脑袋，而是被参数结构倒逼出来的——分类树草案只是起点，**最终形状由你们的实际参数表校准**。

### 2.2 分类编码

`大类-子类-孙类` 三段大写字母码，如 `PA-AMP-IP`（公共广播-广播功放-IP网络功放）、`PRO-SPK-LINE`（专业音响-音箱-线阵）。写入每行产品，作为 AI 检索的主键之一。编码在分类树定稿时冻结，之后只增不改。

### 2.3 归类的判定规则（消灭模棱两可）

1. **按主职能归类**，不按附带功能：带对讲功能的网络功放归"IP网络功放"，不归"对讲终端"；
2. **一物一类**：任何产品只出现在一个末级分类里，绝不重复挂靠；
3. **归不进去的**：先记入 `UNSORTED 待定池`并写明原因，积累到一定数量再决定新增孙类还是并入近邻，不阻塞主流程；
4. **套装/系统级商品**（如"一套无纸化系统"）：拆成单品归类；确实不可拆的，进各大类下的 `KIT 成套系统` 孙类，只填公共列+文本描述。

### 2.4 分类校准流程（拿到参数表后第一步）

① 将全部型号自动映射到分类树草案 → ② 输出映射结果+待定池给你确认 → ③ 按 2.1 硬规矩检查每个孙类的参数结构一致性，该拆的拆、该并的并 → ④ 你签字确认分类树 → 冻结编码，进入字典阶段。

**分类树必须你亲自过目一次**：分类错了后面全部返工，这是整个流程里唯一必须人工把关的闸门。

## 三、怎么定义参数字典

### 3.1 字典表的结构

每个末级分类一张字典表，每行定义一个拆解列，七个字段：

| 字段 | 说明 | 示例 |
|---|---|---|
| 列名 | 中文+单位后缀，全表唯一 | `单通道额定功率_8Ω_W` |
| 类型 | 数值/枚举/布尔/文本 四选一 | 数值 |
| 单位 | 见 3.3 单位基准 | W |
| 允许值 | 枚举列的受控词表；数值列的合理区间（用于校验） | 10–5000 |
| 别名 | 厂家原文里的各种叫法 | 输出功率、RATED POWER、额定输出 |
| 必填级 | 核心（该类产品选型必看）/ 一般 / 可选 | 核心 |
| 说明 | 取值口径 | 立体声模式 8Ω 每声道 |

### 3.2 列类型只有四种（没有第五种）

| 类型 | 硬规则 | 反例（禁止） |
|---|---|---|
| **数值** | 格子里只有纯数字；单位固定在列名里；范围值拆 `_min/_max` 两列 | 格子里写 "350W"、"50Hz-20kHz" |
| **枚举** | 只允许字典列出的受控值；新值须先入字典 | 同列混写 "Class D / D类 / 数字功放" |
| **布尔** | 只允许 `是/否` | "支持"、"√"、"有" |
| **文本** | 兜底，用于确实无法结构化的描述 | 把能拆的数硬塞进文本列 |

### 3.3 单位基准（录入时强制换算）

功率 W ｜ 频率 Hz ｜ 声压 dB ｜ 尺寸 mm ｜ 重量 kg ｜ 屏幕 inch ｜ 机架 U ｜ 亮度 nit ｜ 色温 K ｜ 电压 V ｜ 角度 °(deg)。
例：20kHz → 填 `20000`；1.5kg 与 1500g 全部归 kg。

### 3.4 公共列（全品类一致，约 14 列）

`型号｜产品名称｜大类｜子类｜孙类｜分类编码｜宽_mm｜高_mm｜深_mm｜机架高度_U｜重量_kg｜供电方式｜整机功耗_W｜安装方式`
——支撑跨类检索（如"找 2U 以内、POE 供电的设备"），又不撑爆表。

### 3.5 专属列的产生方式

对每个末级分类：扫描该类全部产品的原文参数 → 出现频率高的参数进字典（核心/一般）→ 只有个别型号才有的参数不单独开列、进`备注说明`→ 输出字典给你过目（可选）。每类专属列预计 10–25 列，与你"追加几列到十几列"的预期一致。

### 3.6 同义词归一表（字典的字典）

全局维护一张别名表（例：`D类 ← Class D/数字功放/数码功放`；`POE ← PoE/网线供电/IEEE802.3af`）。作用：几千个 SKU、多批次录入、甚至换一个 AI 来录，口径不漂移。随录入持续累积。

## 四、怎么拆解（逐行执行的 10 条规则）

1. **原文永不改动**：原始参数列逐字保留，拆解只做"追加"，出争议可回查；
2. **组合值拆散**："2×350W(8Ω)" → `通道数=2`、`单通道额定功率_8Ω_W=350`；
3. **范围值拆两列**："50Hz-20kHz" → `频响下限_Hz=50`、`频响上限_Hz=20000`；"120°×60°" → 水平/垂直两列；
4. **多工况各归各列**：8Ω/4Ω/桥接/定压各自独立列，绝不混在一格；
5. **单位强制换算**成 3.3 基准；
6. **措辞归一**：按同义词表把枚举值收敛到受控词表；
7. **多档取值有口径**：参数随模式变的（如摇头灯不同模式 DMX 通道数 16/24/32），数值列取最大档，全部档位写进备注——口径写进字典说明，全类一致；
8. **缺失就是缺失**：原文没有的留空+`解析状态=缺失`，**绝不编造**；
9. **推断要留痕**：按行业惯例补全的值（如只写"350W"未注阻抗，广播功放按定压口径记）标`推断`+备注写明依据；
10. **冲突要暴露**：原文自相矛盾或与同系列规律不符的，值可先填但标`冲突`+备注摘录矛盾原文，进复核清单。

### 4.1 解析状态列（每行产品最后两列）

`解析状态`：**明确 / 推断 / 缺失 / 冲突** 四值；`备注说明`：自由文本写依据。
原则：**AI 拆出的推断值不得冒充厂家明确标注值**——匹配系统对`推断`按低置信处理。你复核时只需筛`推断`和`冲突`两种，不用逐行看。

### 4.2 完整拆解示例（评审重点，看这里最直观）

**例1 · 专业功放**，厂家原文：
> 额定功率：2×350W@8Ω / 2×550W@4Ω，桥接 1100W@8Ω；频率响应：20Hz-20kHz(±0.5dB)；信噪比：≥103dB；THD：<0.05%；D类；2U

| 拆解列 | 值 | 状态 |
|---|---|---|
| 通道数 | 2 | 明确 |
| 单通道额定功率_8Ω_W | 350 | 明确 |
| 单通道额定功率_4Ω_W | 550 | 明确 |
| 桥接功率_8Ω_W | 1100 | 明确 |
| 功放类型 | D类 | 明确 |
| 频响下限_Hz / 频响上限_Hz | 20 / 20000 | 明确 |
| 信噪比_dB | 103 | 明确（"≥"取标称值，口径入字典） |
| THD_pct | 0.05 | 明确（"<"同上） |
| 机架高度_U | 2 | 明确 |

**例2 · 专业音箱**，厂家原文：
> 12寸低音×1+3寸高音×1，额定 350W，8Ω，灵敏度 98dB，最大声压 124dB，覆盖角 90°×60°，桦木夹板箱体

→ `低音单元尺寸_inch=12`｜`低音单元数量=1`｜`高音单元尺寸_inch=3`｜`高音单元数量=1`｜`额定功率_W=350`｜`额定阻抗_Ω=8`｜`灵敏度_dB=98`｜`最大声压级_dB=124`｜`水平覆盖角_deg=90`｜`垂直覆盖角_deg=60`｜`箱体材质=桦木夹板`——全部`明确`。

**例3 · 歧义处理**，厂家原文：
> 输出功率：650W；连续声压级：121dB

| 拆解列 | 值 | 状态 | 备注 |
|---|---|---|---|
| 额定功率_W | 650 | **推断** | 原文未注明额定/峰值，按行业惯例记额定；若同系列峰值≈2×额定规律不符则改判 |
| 最大声压级_dB | 121 | **冲突** | 原文写"连续声压级"，与本列"最大(峰值)"口径不同；数值先记，待确认是连续还是峰值 |

→ 这两条自动进复核清单，你确认后改`明确`。

## 五、质量控制

1. **规则自动校验**：每类宽表跑脚本检查——数值列无非数字字符、枚举值全在受控词表内、min≤max、数值落在字典合理区间（如灵敏度 80–110dB，出界即标`冲突`）；
2. **抽检**：每类随机抽 10% 与原文人工比对，错一处即全类复查该规则；
3. **复核清单闭环**：`推断/冲突`条目你确认后回写状态，清单清零该类才算交付；
4. **一致性回归**：新批次录入后重跑校验+同义词表比对，防止口径随时间漂移。

## 六、实施计划（分三阶段）

| 阶段 | 内容 | 你需要做的 | 产出 |
|---|---|---|---|
| **P1 试点** | 你发参数表 → 分类映射+待定池 → 选 2 个类（建议专业功放+专业音箱，各几十型号）做完整样板 | ①确认分类树 ②看样板宽表格式是否满意 | 分类索引表 v1、2 张样板宽表 |
| **P2 铺开** | 按签字格式批量处理全线；每完成一个大类交付一批 | 处理各批复核清单（只看推断/冲突） | 全部宽表+字典 |
| **P3 收口** | 全库校验回归、同义词表定稿、交付使用说明 | 终审 | 四件套完整交付 |

先试点再铺开的原因：格式问题在几十行时改是小事，在几千行时改是灾难。

**给我参数表的方式**：Excel/CSV 均可，推到仓库或直接发我；表内乱格式（合并单元格、参数写在一格里）不用预处理，拆解就是我的活。

## 七、这套数据后续怎么被 AI 使用（为什么这样设计）

- 用户问"找一台 4 通道、每通道 500W@8Ω 以上、带 Dante 的功放" → AI 直接在 `PRO-AMP` 宽表按三列过滤，秒出，无需再读原文猜；
- 竞品对标/方案替换 → 同类宽表列对列比较，差异一目了然;
- 字典表+同义词表让任何 AI（不限于我）都能按同一口径读写这套数据——**字典是给机器看的产品知识说明书**，这是比宽表本身更长效的资产。

## 八、需要你拍板的评审点

1. 分类树草案的大类划分是否符合你们的业务线组织（`00_分类树草案` 全文）？

4. 试点先做"专业功放+专业音箱"两个类，还是你指定别的类？
