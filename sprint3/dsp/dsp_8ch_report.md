## 0. 执行摘要（CTO 级别，<200字）

8通道 TDM 真实硬件（竞品拆机确认）使算法链从 Sprint2 的 N=16 降至 N=8 有效处理通道。

**关键结论**：
- **算力裕量** 从 Sprint2 的 26× 提升至 **49×**（30.56 vs 58.11 MMAC/s）[脚本测算]
  - ⚠️ **2026-05-28 P0-2 树形 C 实算纠正（以此为准）**：真实树形 MCPS **8ch=45.7 / 16ch=88.7 MMAC/s**（非纸面 30.56/56.4——漏算分析侧 interp2，低估 ~1.6×）；**裕量 33×(8ch)/17×(16ch)**（非 49×）。本报告正文以下 30.56/49× 为历史纸面值，权威值见 `sprint3/dsp/tree_filterbank.c` + DEC-S3-P0-01。结论方向（裕量充足、远超 ≥10×）不变，仍待 EZKIT cycle 实测。
- **端到端延迟** 12.53ms **不变**——延迟由滤波器组树深/抽头+DMA+DAC决定，与通道数无关；"8通道比16通道快多少"的诚实回答是：**延迟≈不变，算力≈减半**
- **架构约束**：A/B 中心对称串联 → 8路驱动器 = 8个对称元素对 → 只能 broadside 波束整形，**无法电子偏转**，Dolph-Chebyshev 权重落到8个对称对上

## 2. 算力预算（N_CH=8，脚本复用 budget_calc.py）

> 工具调用：`python3 budget_calc.py`（将 `N_CH=16` 改为 `N_CH=8`，其余参数不变）

### 2.1 子带划分（维持 Sprint2 4子带 dyadic 树形）

> **几何更正（DEC-S3-GEOM-01）**：自研基线阵元间距统一为 **d=55mm[L1拆机实测]**（旧值 d=30mm 系视觉估测错误，已作废）。下表 d/λ 与栅瓣按 d=55mm 重算。

| 子带 | 频率范围 | 抽取率 M | 子带率 | 半带 FIR | d/λ 范围 (d=55mm) | 栅瓣 |
|------|----------|---------|--------|---------|---------|------|
| SB0 | 500–1000 Hz | 16× | 3 kHz | 31 tap | 0.080–0.160 | 无 |
| SB1 | 1k–2k Hz | 8× | 6 kHz | 63 tap | 0.160–0.320 | 无 |
| SB2 | 2k–4k Hz | 4× | 12 kHz | 63 tap | 0.320–0.641 | 无（< 1.0）|
| SB3 | 4k–8k Hz | 2× | 24 kHz | 63 tap | 0.641–1.282 | **6.2–8kHz 段有栅瓣**[L1/几何] |

结论（d=55mm[L1拆机实测]）：
- **500Hz–5kHz 安全**（d/λ < 1.0，无栅瓣）。
- **6.2–8kHz 出现栅瓣**：半波长判据 d=λ/2 → 3118Hz，严格判据（broadside, d=λ）→ 6236Hz；6.24kHz 起栅瓣峰达 **0dB（与主瓣等高）**，带外泄漏 78–89%[L2 仿真]。
- 应对：强指向规格上限由 8kHz **降级为 6kHz（对内）/ 5kHz（对外保守）**（方向1 规格降级，CTO 采纳）。SB3 上半段（6.2–8kHz）落在降级后规格之外，不影响主指向频段。
- 判据依据见 audit §9 grating-lobe criterion + DEC-S3-GEOM-01。
> 措辞红线：d=55mm 为拆机实测值[L1]；上述 d/λ、栅瓣峰、泄漏率为仿真/几何推算[L1/L2]，非实测指向性。

## 3. 群延迟分析——"8通道比16通道快多少"的诚实回答

> 工具调用：`python3 latency_calc.py`（参数与通道数无关，直接复用）

### 3.1 延迟构成（dyadic 树形，63-tap 半带 FIR，SB0 最长路径）

| 项目 | 延迟 (ms) | 与通道数关系 |
|------|-----------|------------|
| 树形滤波器组群延迟（分析+合成，SB0） | 9.042 | **无关** |
| DMA ping-pong（2帧 × 64smp @ 48kHz） | 2.667 | **无关** |
| DAC 群延迟（ADAU1966A，36smp @ 48kHz）[待核实] | 0.750 | **无关** |
| 分数延迟 FIR 群延迟（3.5smp） | 0.073 | **无关** |
| **端到端总延迟** | **12.53 ms** | — |

### 3.2 诚实回答 CTO 的问题

> **"8通道比16通道在延迟上快多少？"**

**回答：快 0 ms。延迟完全不变，仍为 12.53 ms。**

原因：端到端延迟由以下因素决定，**全部与通道数 N_CH 无关**：
1. **树形滤波器组群延迟**：由半带 FIR 抽头数（63 tap）和树级数（3级）决定，计算公式为 `Σ (N-1)/2 / fs_level`，与通道数无关；
2. **DMA 延迟**：由帧大小（64 samples）决定；
3. **DAC 延迟**：由 ADAU1966A 内部数字滤波器决定；
4. **分数延迟 FIR 群延迟**：由 FIR 阶数决定。

通道数减少影响的是**算力负载**（MMAC/s 近似减半），**不影响任何延迟项**。

延迟已于 Sprint2 经 `latency_calc.py` 脚本核实，在 12.53 ms 下满足 <30 ms 规格（DEC-S2-spec 放宽后），对博物馆/车站/商场单向播放场景无感知影响。

## 4. 8驱16（A/B 中心对称串联）的算法适配

### 4.1 物理架构说明

```
阵元物理编号:  A1  A2  A3  A4  A5  A6  A7  A8 | B8  B7  B6  B5  B4  B3  B2  B1
                ←── 8个 A 单元 ────────────────|──────── 8个 B 单元 ───────────→
                                              ↑ 阵列物理中心

驱动信道:      Ch1  Ch2  Ch3  Ch4  Ch5  Ch6  Ch7  Ch8
串联映射:      A1=B16, A2=B15, A3=B14, A4=B13, A5=B12, A6=B11, A7=B10, A8=B9

对称对定义（广义中心对称）:
  对 1:  元素 A1 和 B16（关于中心对称，距中心最远）
  对 2:  元素 A2 和 B15
  ...
  对 8:  元素 A8 和 B9（距中心最近）
```

**激励规则**：同一驱动通道的两个单元接收**完全相同**的信号（串联 → 同幅同相）。

### 4.2 对称约束的全部影响

| 影响点 | 描述 | 结论 |
|--------|------|------|
| **波束偏转** | 中心对称结构的 AF（阵因子）为纯实数偶函数，仅支持 broadside（0°），无法产生不对称的相位延迟偏转 | **只能 broadside，无法电子偏转** |
| **权重自由度** | 16个单元 → 8个独立权重（对称对共享一个权重），波束整形自由度从 N=16 降至 **N=8** | 独立权重数 = 8 |
| **波束指向灵活度** | **无**。只能通过8个对称对权重做旁瓣整形（如 Dolph-Chebyshev），不能非对称偏转 | 参见下方说明 |
| **Dolph-Chebyshev 适配** | N=16 D-C 权重集是关于中心对称的（奇数位=对称分布），正好与8对串联映射对应；直接取 D-C 权重的前8个（= A1…A8 端），后8个（B16…B9 端）由物理串联自动复现 | D-C 权重无损适配 |
| **旁瓣控制** | 仍能通过 D-C 权重实现旁瓣整形（-20dB SLL 目标保持），但不能在整形同时偏转主瓣 | 整形能力保留 |
| **阵因子** | `AF(θ) = 2 Σ_{n=1}^{8} w_n cos[(2n-1)πd sinθ/λ]`（偶函数，broadside 对称） | 公式确认 |

### 4.3 残余波束指向灵活度

**结论：零灵活度偏转，但有波束宽度整形灵活度。**

可做的：
- 通过改变 D-C 权重的 SLL 参数（ε 值）调节主瓣宽度（更宽 = 更低 ε；更窄 = 更高 ε）[估算]
- 各子带独立整形（低频宽波束/高频窄波束，符合指向频段 ≥1kHz 策略）

不可做的：
- 电子波束偏转（无法实现非对称相位延迟）
- 双波束并行（无足够自由度）
- MVDR 自适应抑制特定角度干扰（需要非对称权重）

> 如需波束偏转，**必须修改硬件**（取消 A/B 串联，改为 16ch 独立驱动），这是硬件架构约束，算法无法绕过。

### 4.4 Dolph-Chebyshev 权重在8对结构上的适配

Sprint2 的 N=16 D-C 权重（旁瓣电平 -20dB）是关于阵列中心对称分布：
```
w = [w1, w2, w3, w4, w5, w6, w7, w8, w8, w7, w6, w5, w4, w3, w2, w1]
      ↑─────────── A1-A8 端 ──────────↑───── B16-B9 端 ─────────────↑
```

A/B 串联后，硬件天然实现了后8个权重的复现，算法只需输出前8个 `[w1…w8]`，无需任何修改即可维持原 D-C 指向性能。

**分数延迟适配**：broadside 时所有通道分数延迟 = 0（整数延迟 τ_n = 0），分数延迟 FIR 退化为单位冲激，**算力进一步降低**（波束形成核心算力 3.24 MMAC/s 中大部分为极小值）[估算]。

```c
/* 注意：[待核实 U4] 竞品真实 BCLK/FSYNC/slot 配置需逻辑分析仪实测 */
```

### 5.2 算法链（8ch broadside-only）

```
数字节目源（单声道节目 @ 48kHz）
        │
        ▼
┌─────────────────────────────────────┐
│   dyadic 树形分析滤波器组（3级）       │  分析 SB0/SB1/SB2/SB3
│   N_CH=8，63-tap 半带 FIR             │  8ch 并行
└──────────────┬──────────────────────┘
               │ 4 子带 × 8 通道
               ▼
┌─────────────────────────────────────┐
│   8对对称对加权求和（broadside-only）  │
│   w[k] = D-C 权重（第 k 对，k=1..8）  │
│   分数延迟 FIR: broadside 时 τ≈0      │
│   加权求和: y_sb = Σ w[k] × (A_k+B_k)│  [A_k、B_k 同信号，= 2×w[k]×A_k]
└──────────────┬──────────────────────┘
               │ 4 子带 × 1 通道
               ▼
┌─────────────────────────────────────┐
│   dyadic 树形合成滤波器组（3级）       │  重建 48kHz 单声道
└──────────────┬──────────────────────┘
               │ 1 通道 @ 48kHz
               ▼
┌─────────────────────────────────────┐
│   子带增益均衡（可选，各子带 ×G_sb）   │  补偿喇叭频响偏差
│   DRC/限幅器（保护功放）               │
└──────────────┬──────────────────────┘
               │
               ▼
         8ch TDM 输出
   （同一信号广播到 8 个驱动通道）
          [注 1]
```

> [注 1] broadside-only 时，所有 8 个通道输出信号**相同**（仅幅度因 D-C 权重不同），可视为对同一处理结果的幅度加权分发。

## 6. 定点化要点（SHARC+，针对8ch结构）

ADSP-21569 支持原生 32/40bit 浮点，**量产前无需强制定点化**。开发阶段建议：

| 阶段 | 格式建议 | 原因 |
|------|---------|------|
| EZKIT 原型 | 32bit float | SHARC+ 原生，移植最容易，调试快 |
| 量产评估 | 保留 float 或迁移 Q31 | 视成本/功耗需求决策（Gate-2 后） |

**关键风险（定点化时）**：
- FIR 累加溢出：8ch × 63tap MAC，Q31 累加器需检查最坏情况（参见 FP_001 陷阱）
- 半带 FIR 系数量化：Q15 格式，`(N-1)/2 = 31`，通带波纹需 < 0.1dB
- D-C 权重精度：Q15 已足够（旁瓣 -20dB 对应动态范围约 60dB，Q15 SNR=90dB 绰绰有余）
- **无 IIR**，无极限环风险（FP_002 不适用）

## 8. 待 CTO 决策项

1. **broadside-only 限制是否可接受？** 如需波束偏转（如偏转 ±15°），必须将硬件改为 16ch 独立驱动（影响转接板设计，闸门 Gate-2 前决策）
2. **算力裕量 49× 是否足以支持后处理模块扩展？** （EQ、DRC、多路输入路由等）— 当前 1469 MMAC/s 剩余，可支持大量后处理，建议确认后处理需求清单
3. **喇叭匹配确认**：6项未知量中 U3（喇叭阻抗/功放增益）确认后，需重新校核 D-C 权重幅度归一化是否触发功放限幅
