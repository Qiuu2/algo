## 0. 结论速览（给读者）

| 节点 | int64 累加器本身 | `>>15 → Q31` 回量化 | 判定 |
|------|------------------|---------------------|------|
| ①半带 FIR MAC | 安全，headroom ≈ **16.2 bit** [L3] | **会溢出，最坏 1.73× 满量程** [L3] | **需处理**（中间饱和 / Q23 输入预留 headroom） |
| ②8ch 加权求和 | 安全，headroom ≈ **17.0 bit** [L3] | 安全，最坏 **0.9998× 满量程**（恰不溢出） [L3] | **桌面可关闭**（1/N 归一化保护到位） |

**真正的风险点不是 int64 累加器溢出**（两节点都有 16-17 bit 富余），
**而是节点①的 `acc>>15 → Q31` 回量化** —— 半带核 Σ|h| = 1.73 > 1，
最坏激励下回量化结果达 1.73× Q31 满量程，超出 int32 范围；
而代码 `return (int32_t)(acc >> 15)` **无饱和**，溢出即环绕（符号翻转）。

无 LOCKED 依据动摇，无假性完成。**不触发 ESCALATE。**（详见第 6 节）

## 1. 最坏激励定义（worst-case excitation）

对任一 `acc += coef_Q15 × state_Q31` 形式的累加链，最坏幅度为：

```
max|acc| = Σ_k |coef_k(Q15)| × full_scale_state(Q31)
```

- **满量程状态**：`state` 为 Q31，full_scale = `2^31 − 1 = 2147483647`（取正满量程；负满量程 −2^31 对称，幅度同阶）。
- **同相相加**：第 k 个 state 取与 `coef_k` 同号的满量程值，使每个乘积项同号叠加 → 累加幅度最大化。
- **系数取真实 Q15 值**：节点①用半带核的实际 Σ|h_Q15|；节点②用 DAS 权重 `w = 32767/N`（Q15）。

这是**解析上界**：任何实际输入（含满量程方波/扫频）都不会超过此值。

## 2. 节点①：半带 FIR MAC（`tree_filterbank.c:40-52`）

### 2.1 节点定义
```c
for (k=0; k<N; k++) {
    rd = ...;
    acc += (int64_t)g_hb[k] * (int64_t)s->state[rd];   /* Q15 × Q31 → Q46 */
}
return (int32_t)(acc >> 15);                            /* Q46 >>15 → Q31，★无饱和★ */
```
- N = `TFB_HB_TAPS = 63`（`tree_filterbank.h:42`）
- 系数 `g_hb[k]`：63-tap Kaiser β=6 半带核，`firwin(63, 0.5, kaiser β=6)` 后 DC 归一化=1，再 Q15 量化
  （定义见 `tree_verify.c:40-60` `make_halfband()`；与 `pf4/q15_stopband_sim.py:73-78` 同一原型）。

### 2.2 真实系数统计（可复现）[L3]
由上述定义数值重建（`numpy.i0` Kaiser + sinc，与 C 端 `i0()` 一致）：

| 量 | 浮点 | Q15 |
|----|------|-----|
| Σ coef（DC 增益） | 1.000000 | 32765（≈ 32768 = unity） |
| **Σ\|coef\|** | **1.730798** | **56717** |
| max\|coef\| | 0.500030 | 16385 |

> 注意：**Σ\|h\| = 1.73 > 1**。半带核中心抽头 ≈ 0.5、相邻抽头正负交替且幅值不小，
> 绝对值和远大于 DC 增益 1 —— 这正是回量化溢出的根因。

### 2.3 累加器（int64）headroom
```
max|acc| = Σ|coef_Q15| × (2^31 − 1) = 56717 × 2147483647 = 1.218e14
         = 2^46.79  →  47 bit 幅度
```
int64 可用幅度位 = 63 bit（1 符号位）。
**headroom = 63 − 46.79 ≈ 16.2 bit** [L3] → **int64 累加器永不溢出，安全。**

### 2.4 `>>15 → Q31` 回量化判定 ★风险点★
```
result = max|acc| >> 15 = 1.218e14 / 32768 ≈ 3.717e9
Q31 满量程 = 2^31 − 1 = 2.147e9
ratio = 3.717e9 / 2.147e9 = 1.7309   →  超 Q31 满量程 73%
```
等价于：半带输出最坏增益 = Σ|h|/DC = 1.7309×（对应一个全幅、与抽头符号完全对齐的对抗性宽带输入）。

- 代码 `return (int32_t)(acc >> 15)` **无饱和、无钳位** → `3.717e9` 超 INT32_MAX（2.147e9），
  C 中 int64→int32 窄化为**实现定义/环绕**，最坏出现**符号翻转**（正峰变大负值），产生爆音/毛刺。
- **下游放大**：`hb_interp2()`（`tree_filterbank.c:75,78`）对该 Q31 结果再 `×2`（内插增益补偿），
  最坏复合 = 2 × 1.73 = **3.46× 满量程**；且 `out` 为 int32，二次溢出无保护。
- 合成端 `a2p[i]=up3[i]+sb1[i]` 等（`:143,147,151`）为 int32+int32 相加，
  各项近满量程时additive 可达 2× → 直到 `beamformer.c:320-323` 最终 Q23 钳位才有饱和（已太晚，中间已环绕）。

### 2.5 实际信号 vs 对抗激励（量化区分）
- 1.73× 是**对抗性上界**（要求输入与全部 63 抽头符号逐点对齐，物理上需特制宽带信号）。
- 半带 LP 通带增益=1，对**带限正弦**输出 ≤ 1×（仅 Gibbs/过冲 ~ 数 %）。
- 但**满量程方波 / 多音 / 阶跃**等真实可遇到的激励会逼近该上界的相当比例 → **不能仅靠"实际信号一般不溢出"放过**。

## 3. 节点②：8ch 加权求和（`sprint2/dsp/cces_template/src/beamformer.c:266-293`）

### 3.1 节点定义
```c
int64_t acc = 0;                                  /* Q46 */
for (ch=0; ch<ARRAY_NUM_ELEMENTS; ch++) {
    ...
    acc += (int64_t)pBf->weight_re[sb][ch] * (int64_t)fd_out;   /* Q15 × Q31 → Q46 */
}
subband_out[sb][s] = (int32_t)(acc >> 15);        /* Q46 >>15 → Q31，无饱和 */
```
- 权重 `w_re = 32767 / N`（Q15，`compute_weights()` `:132`）—— **关键：1/N 归一化已内建。**
- `fd_out` 为 Q31 满量程激励（上界）。

### 3.2 两个口径（代码现状 vs LOCKED 目标）
代码 `config.h:56` 现为 **`ARRAY_NUM_ELEMENTS = 16`**；任务/LOCKED d55 基线为 **8ch 对称 broadside**。
两者结论一致（归一化各自抵消），均列出 [L2 代码常量 / L3 解析]：

| 口径 | N | w=32767/N (Q15) | max\|acc\|=N·w·(2³¹−1) | acc bit | acc headroom | `>>15` 结果/FS |
|------|---|-----------------|------------------------|---------|--------------|----------------|
| 代码现状 | 16 | 2047 | 7.033e13 | 2^46.0 | **17.0 bit** | **0.99951×** |
| LOCKED 8ch | 8 | 4095 | 7.035e13 | 2^46.0 | **17.0 bit** | **0.99976×** |

### 3.3 判定
- **int64 累加器 headroom ≈ 17.0 bit** [L3] → 永不溢出，安全。
- **`>>15 → Q31` 回量化：最坏 0.9995–0.9998× 满量程，恰好不溢出** [L3]。
  因为 `N·w = N·floor(32767/N) ≤ 32767 < 32768`，等效增益 ≤ 0.9998 < 1。
- **结论：节点② 安全，无需额外饱和**（1/N 整数除法向下取整反而提供了微小余量）。
  边界注意：若未来权重改为非归一化（如 Dolph 加权未做 1/N 缩放、或多波束叠加），此余量消失 → 需重评。

## 5. 建议（按优先级）

**节点①（需处理，P1）**：回量化 `(int32_t)(acc >> 15)` 缺饱和，对抗激励下溢出 1.73×。三选一：
1. **加中间饱和（推荐，最小改动）**：把 `return (int32_t)(acc >> 15)` 换成饱和窄化
   `acc>>15` 超 `[INT32_MIN, INT32_MAX]` 时钳位。代价：对抗输入下产生软削波（优于符号翻转爆音）。
   `hb_interp2` 的 `×2` 同样补饱和。
2. **输入预留 headroom**：约定半带链路输入不超 Q31 的 **−4.8 dB**（1/1.73），把 1.73× 上界压回 ≤1×。
   实现简单（输入级一次右移/缩放），代价 ~0.8 bit 动态范围 + 需全链路约定。
3. **改 Q 格式留整数位**：半带输出用"Q29 + 2 整数位"承载 ≤4× 范围，最后统一缩放。改动大，不推荐。

> 倾向 **方案 1 + 方案 2 组合**：实时核加饱和兜底（防爆音），系统级约定输入 headroom（防止常态削波）。
> 注：实测层面，`tree_verify.c` 用的是 PR 单位增益激励（DC=1），所以现有桌面 PR 验证 **不会触发**此溢出 —— 这正说明它是一个"对抗激励才暴露"的潜伏点，需显式处理而非依赖现有用例。

**节点②（无需处理）**：1/N 归一化使等效增益 ≤0.9998，恒不溢出。
唯一约束：**保持权重归一化**；若引入 Dolph 加权或多波束叠加导致 Σw > 1，须重做本分析。
