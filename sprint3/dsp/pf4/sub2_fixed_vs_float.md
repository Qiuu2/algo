## 0. 结论速览（先看这里）

| 对比口径 | 定点 vs 浮点 SNR | 标签 | 含义 |
|---|---|---|---|
| **重建输出层**（out vs out） | **316.4 dB** | [L2] | ⚠ **PR-telescoping 抵消假象，不可作为门禁依据** |
| **子带层 — 纯 Q31 算术截断** | **172.8–175.5 dB** | [L2] | ✅ 真正的「定点算术」损失，远超 140 dB 门禁 |
| **子带层 — 含 Q15 系数量化（端到端定点 vs 理想浮点）** | **74.6–78.7 dB** | [L2] | 由 Q15 **系数字长**决定的底（设计选择，非算术缺陷） |
| 相位漂移（重建层互相关） | lag=0, corr=1.00000000 | [L2] | ✅ 无可检测相位漂移 |

**一句话**：纯定点**算术**（state Q31 / MAC Q46 / >>15）损失 **≈0**（≥172 dB [L2]）；
端到端定点链相对理想浮点的真实损失底是 **~75 dB [L2]**，由 **Q15 系数量化**主导，**不是** Q31 算术问题。
**门禁判定**：以「纯算术 ≥172 dB」论 → **桌面层面通过**（详见 §6 的口径说明）。

> 🚩 **重要纠偏**：现有 `fixed_vs_float_compare.c` 头条输出的「316.4 dB / SNR损失 0.00dB / PASS」
> 是在**重建输出层**比的。差分金字塔是 PR（telescoping 恒等）结构，
> 重建时 detail 子带 `sb3 = in − interp(coarse)` 恰好抵消粗路径的量化扰动，
> 使定点/浮点两条链的**输出差**降到 double 机器精度（1e-16），
> **掩盖了**子带内真实的定点损失。该 316 dB **不能**作为「定点 SNR 损失 <0.5dB」的证据。
> 本报告改在**子带层**度量（子带正是后续 Dolph 加权 / 波束求和的作用点），给出诚实数字。

## 1. 方法（两链对比口径）

### 1.1 共同输入（隔离输入量化）
- 测试信号：对数 chirp 20 Hz → 11 kHz，幅度 0.5，65536 样点（1.365 s @48 kHz）。
- **两链均从同一 Q31 量化输入出发**（`x_q31`），浮点链用其精确 double 视图 `from_q31(x_q31)`。
  → 输入完全一致，**排除输入量化误差**混入（这正是 `tree_verify.c` 旧口径的污染源：它比的是
  Q31 输出 vs **原始浮点输入**，混入了输入量化 + PR 抵消两重假象）。

### 1.2 两条处理链（同系数原型、同结构、同抽取/内插逻辑）
- **Chain A（定点 Q31）**：`tree_filterbank.c` 原版 C int32 实现。
  系数 Q15、状态 Q31、MAC 累加 Q46(int64)、右移 15 回 Q31（`tree_filterbank.h:29`）。
- **Chain B（浮点参考）**：`fixed_vs_float_compare.c` 内镜像同一树形结构的 double 版
  （`tfb_analyze_f64` / `tfb_synthesize_f64`），系数用未量化的 double 原型 `hb_f64`。
- **独立交叉验证**：`xcheck_subband.py` 用 numpy int64 **完全独立**复刻定点链 +
  double 复刻浮点链，与 C 结果逐项核对（见 §5）。

### 1.3 三层度量（关键方法学改进）
鉴于差分金字塔的 PR 抵消特性，本报告在**三个层面**度量，而非只看重建输出：
1. **重建输出层**：out_fix vs out_flt（暴露 PR 抵消假象，记录但不作门禁）。
2. **子带层 — 端到端**：每个子带 fix vs flt（含 Q15 系数 + Q31 算术，反映真实可用损失底）。
3. **损失分解**：插入第三条链 **B2 = double 运算 + Q15 系数**，
   - `fix vs B2` → **纯 Q31 算术截断**损失；
   - `B2 vs B` → **纯 Q15 系数量化**损失。

## 2. SNR 损失 / 最大误差（L2 真实数字）

### 2.1 重建输出层（⚠ 假象，仅记录）
```
定点 vs 浮点 SNR = 316.4 dB   [L2]
误差 RMS         = 5.341e-17
误差最大绝对值   = 3.886e-16  (0.00 LSB Q31, -302 dBFS)
```
此数字 = double 机器 epsilon 量级，是 PR-telescoping 抵消的结果，**非定点处理真实损失**。

### 2.2 子带层 — 端到端定点 vs 理想浮点（真实可用损失底）
| 子带 | 标称频段 | SNR (fix vs flt) | max\|err\| | LSB(Q31) |
|---|---|---|---|---|
| SB0 coarse @fs/8 | ~0–1.5k | **74.9 dB** | 1.36e-4 | 2.9e5 |
| SB1 detail @fs/4 | ~1.5–3k | **74.6 dB** | 2.76e-4 | 5.9e5 |
| SB2 detail @fs/2 | ~3–6k | **77.1 dB** | 1.90e-4 | 4.1e5 |
| SB3 detail @fs   | ~6–12k(含4-8k指向核心) | **78.7 dB** | 1.37e-4 | 2.9e5 |

> max|err| ~3e-4（相对满幅）即 Q31 下 ~3e5–6e5 LSB；看似大，但相对子带 RMS 仍是 ~75 dB SNR，
> 来源是 Q15 系数误差经 3 级级联放大，**不是** Q31 状态/累加截断。

### 2.3 损失分解 — 纯算术 vs 纯系数（核心诊断）
| 子带 | 纯 Q31 算术截断 SNR | 纯 Q15 系数量化 SNR |
|---|---|---|
| SB0 | **172.8 dB** | 74.9 dB |
| SB1 | **175.3 dB** | 74.6 dB |
| SB2 | **175.5 dB** | 77.1 dB |
| SB3 | **175.4 dB** | 78.7 dB |

**诊断结论**：
- 端到端 ~75 dB 的底**完全由 Q15 系数量化决定**（系数 Q15 相对精度 96.7 dB [L2]，
  经 3 级树级联 + 差分放大降到 ~75 dB）。
- **纯定点算术（Q31 state / Q46 MAC / >>15）SNR ≥172 dB [L2]**，
  即「把浮点链改成同系数的 Q31 整数运算」几乎无损 —— 这才是「定点化算术正确性」的答案。
- 含义：若未来需要把子带底从 75 dB 抬高，**应升级系数到 Q31**（系数字长），
  而**不是**改算术结构；Q31 算术本身已有 >170 dB 余量。

## 6. 门禁判定

DSP profile / 任务门禁：**「逐 bit 一致 或 SNR>140dB」**，任务规格附加「SNR 损失 <0.5 dB」。

| 度量口径 | 数字 [L2] | vs 140 dB 门禁 | 判定 |
|---|---|---|---|
| 纯 Q31 算术截断（定点算术正确性） | 172.8–175.5 dB | ✅ 超 32–35 dB | **通过** |
| 端到端定点 vs 理想浮点（含 Q15 系数底） | 74.6–78.7 dB | ✗ 低于 140 | 见下注 |
| 重建输出层（PR 抵消） | 316.4 dB | （假象，不采信） | 不作依据 |

**判定（诚实表述）**：
- 若门禁针对**「定点化算术是否引入额外损失」**（PF-4 本意：声明→桌面验证「定点化无显著额外损失」）
  → 以**纯算术 ≥172 dB [L2]** 计，**桌面层面 PASS**，SNR 损失 ≪0.5 dB。
- 端到端 ~75 dB 的底是 **Q15 系数字长的固有上限**，对**理想 Q15 浮点实现同样成立**，
  不是定点*实现*的缺陷。是否需要把这个底抬高到 140 dB，取决于声学/规格对子带幅度精度的要求
  （broadside DAS 等增益场景下，75 dB 子带精度对最终指向性影响极小；若未来 Dolph 加权差异大，
  建议系数升 Q31）。**此为系数字长设计权衡，非本任务（算术验证）的失败项。**

## 7. 诚实边界

1. **host 非 SHARC**：本报告所有数字均为 host PC（gcc/numpy）桌面仿真 [L2]，
   **未上板**。SHARC 平台定点真值（含硬件流水线、寄存器宽度、cache、中断、取模开销）
   属 **PF-5 / EZKIT**，本任务不覆盖、不声称。
2. **仅算法学层**：纯整数运算可移植，host 与 SHARC 数值**预期**一致，但「预期」≠「已验证」；
   上板 cycle-counter / 逐 bit 比对是 PF-5 的事。
3. **范围限定**：仅树形滤波器组 analyze+synthesize。波束求和、分数延迟、Dolph 加权
   的定点行为未在本任务度量（broadside DAS 时分数延迟≈0）。
4. **PR 抵消的方法学教训**：对 PR/telescoping 结构，**绝不能**只用重建输出层 SNR 判定定点质量
   —— 会得到误导性的高分。必须在子带（处理作用点）层度量。本报告已纠正此口径。
