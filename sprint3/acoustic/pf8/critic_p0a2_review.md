## 1. 方向2 命门 — 独立验证（self numpy，未抄报告脚本）

独立实现 N=16/d=55mm/broadside，**自取 scipy.signal.windows.chebwin(16, 20)** 归一化加权，
c=343 显式，θ∈[−90°,90°]，360001 点（0.0005° 分辨率），主瓣外（|θ|>5°）找峰：

```
f=6236Hz: 栅瓣峰 = −0.000 dB @ −90.0°  (d/λ=1.000)
f=6500Hz: 栅瓣峰 = −0.000 dB @ −73.6°  (d/λ=1.042)
f=7000Hz: 栅瓣峰 = −0.000 dB @ −63.0°  (d/λ=1.122)
f=8000Hz: 栅瓣峰 = −0.000 dB @ −51.2°  (d/λ=1.283)
```

**结论：完整复现报告 §3 验证A。** 施加完整 Dolph-Cheby −20dB 加权后，栅瓣峰仍精确 = 0.00dB
（与主瓣等高），栅瓣角与报告/量化交付2 [L2] 逐点一致。

**物理解释独立认同**：栅瓣源于空间采样混叠，d≥λ 时 sinθ_gl=mλ/d 处满足与主瓣**相同的相干叠加条件**；
幅度加权 wₙ 对主瓣与栅瓣**对称作用**，无法降低栅瓣相对电平。这是采样定理硬约束，与加权策略无关。
→ 方向2"纯阵因子加权不能补救" **[L2 独立复算证实]**，结论成立，非纸面照抄。

阵元指向性抑制量亦独立复算一致：51.2°→cos −4.1dB/cos² −8.1dB；63°→−6.9/−13.7dB；
73.6°→−11.0/−22.0dB。趋势认同：8kHz 栅瓣最靠 broadside（51°），阵元抑制最弱——最该抑制处帮忙最少。

## 2. 方向1 判断审查（强指向上限降 6kHz）

**审查结论：成立，无逻辑漏洞。**

- 与 CTO 已锁决策一致：[directivity-band-requirement] 明确"仅需 ≥1kHz 指向、语音可懂度驱动"，
  本判断不越界、不与 CTO 决策矛盾。
- 物理链条自洽：6.24kHz 才入等高栅瓣，1–4kHz（d/λ=0.16–0.64）副瓣维持 −20dB 底、强指向完整，
  语音可懂度主能量（辅音/共振峰 2–4kHz）全覆盖。牺牲 6.2–8kHz 仅损"定向感/音色亮度"，非可懂度。

- 诚实点：报告未夸大——明确指出 6.2–8k 准全向会向 ±51–74° 邻区泄漏高频（分区广播串扰），
  并给信号链缓降兜底，未把代价说成"零"。逻辑闭环。
