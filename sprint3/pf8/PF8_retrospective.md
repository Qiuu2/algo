## 2. 关键失误点（诚实，含 Critic 自省）

### 失误点 A：d=30 被 LOCKED 时，Critic 为什么没拦？

**诚实归因——不是 Critic 当时失职，而是当时无武器可用：**
- **POLICY-PROV-001 当时尚未建立**（该制度缘起更晚的 PF-1 算力审计，2026-05-28 才生效）。Sprint 2 锁 d=30 时，没有"数字必须标来源等级"的强制要求，更没有"L0 严禁 LOCKED"的红线。
- **无几何专项门禁**：Critic 当时的 checklist（声学/DSP/硬件/结构…）中**没有"几何尺寸 LOCKED 前须 L1 实测"这一项**。一个目测来的间距值，在当时的流程下可以平滑地进入 LOCKED 状态而不被任何检查项拦下。
- **无 L0 概念**：即便有人模糊意识到"d=30 是看出来的"，当时的分级最低只到 L4（占位/行业参考）。"纯目测"被默认塞进 L3/L4，**可信度被系统性高估**——读者无从知道这个数比"占位假设"还不可靠。

> **Critic 自省**：A 是"制度真空"型失误，**追责到个人是错误的归因**；但 Critic 必须承认：质量守门的职责包含"在制度缺位处主动补门"。当时若 Critic 把"几何尺寸是不是测出来的"作为对抗式提问（soul §3.2 边界条件追问的应有之义），有机会更早暴露。这一反省已固化为 **C6 几何门禁**。

### 失误点 B：拆机实测 d=55 后，为什么没自动触发 DEC-S2-006 重审？

**三因叠加：**
1. **无强制重审机制**：项目当时没有"L1 实测与 LOCKED 冲突 → 强制重审被冲突的 LOCKED 决策"这条规则。拆机得到 d=55 后，它被当作"竞品的一个事实数据"记录下来，**而没有人被流程要求回头去质问"那 DEC-S2-006 锁的 d=30 还成立吗"**。冲突客观存在，却没有触发器。
2. **"并存"看似合理（最隐蔽的陷阱）**：d=30 当时挂的是"自研基线"，d=55 是"竞品实测"。两者分属不同对象，于是"自研用 d=30、竞品是 d=55"听起来**逻辑自洽、毫无违和**——掩盖了一个根本事实：**d=30 自始至终就是个错误的 [L0] 估测值，根本不存在"自研真的选了 30mm"这回事**，它只是把竞品看走眼的产物。把一个"错值"包装成"我方的设计选择"，是合理化最危险的形态。
3. **快速验证线 KPI 错位**：拆机隶属"快速验证线"，其阶段目标是"拿到 6 项未知量真值"。拿到 d=55 即算交付完成、KPI 达成；**"回头校验自研主线 LOCKED 决策"不在该线的目标函数里**——组织分工的缝隙让冲突从指缝漏过。

### 失误点 C："两条几何并存"为何能长期不被质疑地流通？

- **没有单一事实源（single source of truth）的几何**：自研与竞品两套几何各自被引用，下游文档（PRD、工装、BW 对标）各取所需，**没有任何节点负责"几何一致性"的横向核对**。
- **并存降低了"摩擦"**：撤销一个 LOCKED 决策需要触发重审、惊动 CTO、回改下游——而"并存"零摩擦。**阻力最小路径天然偏向掩盖而非纠错**，这正是为什么必须用制度强制（铁律四"禁并存"）来对抗人性/流程惰性。
- **Critic 跨域一致性检查的盲区**：Critic skill §9 有"acoustics↔structure 腔体体积一致"等接口检查，但**没有"阵列几何参数全项目唯一"这一条**。d 这个参数横跨声学/DSP/结构/PRD，恰恰是最需要唯一性校验的，却落在了检查网格的缝里。

## 3. 制度补丁如何防复发（逐条映射到失误点）

POLICY-PROV-001 升 v1.2，三处新增，**精确对位**上述三个失误点：

| 制度补丁 | 内容 | 堵住的失误点 | 机理 |
|---------|------|------------|------|
| **L0 等级（第 1 部分 + L0 铁律）** | 新设"目测/视觉估算"L0，置于 L4 之下；**L0 严禁用于任何 LOCKED 决策**（不止不可逆，含强约束/几何锁定）；L0 进设计输入必须先升级 L1 | **失误点 A** | 给"纯目测"一个最低可信度身份证。d=30 这种 [L0] 值在新制度下**根本进不了 LOCKED**——在源头被拦，无需事后撤销。 |
| **铁律四（第 4 部分）** | L1 实测值与已 LOCKED 决策冲突 → **自动触发该 LOCKED 决策强制重审，严禁"并存/折衷/两条路线"合理化**；发现者须立即上报 PM+CTO | **失误点 B + C** | 把"冲突→重审"从"靠人自觉"变成"强制触发器"；并直接点名"禁并存"，封死 PF-8 用过的那条合理化路径。 |
| **Critic C6 几何门禁（第 7 部分）** | 几何/物理尺寸参数 LOCKED 前必须有 L1 实测证据（L0 目测/L3 估算作几何 LOCKED 依据=违规）；并查"有无 L1 与 LOCKED 冲突却未重审" | **失误点 A + B（Critic 自身补门）** | 把 A、B 的预防变成 Critic **每次评审的强制 BLOCKER 检查项**，写进 checklist——补上当年"制度真空 + checklist 盲区"。 |
| **CTO 硬约束 SC-S3-GEOM-01** | "d=55 不可逆性以 PRD 不新增 6–8kHz 强指向硬需求为前提；涉之自动触发 d 重议，无 PM 自治权，永久产品边界" | **失误点 C 的未来变体** | 防止未来需求漂移重新撬动几何而无人重审——把"几何重议"的触发条件提前钉死为 CTO 级闸门。 |

**一句话**：A 在**源头**堵（L0 进不了 LOCKED）、B 在**冲突时**堵（铁律四强制重审）、C 在**流通中**堵（禁并存 + C6 唯一性 + SC 永久边界）。三道防线分别对应"进入—冲突—流通"三个阶段。

## 4. 与教训①②③根因对照（同根，PF-8 多一维）

| 教训 | 病灶数字 | 病的类型 | 根因 |
|------|---------|---------|------|
| ① 半角误读（LESSON-008） | BW 14.63°（半角当全角，且一度被当"MATLAB 实测"） | **标注问题**（口径错 + 来源误信） | 数字无身份证 + 未独立复核 |
| ② 纸面当实测（LESSON-007 / PF-1） | 算力裕量 27×/49°（[L3] 纸面 LOCKED 芯片） | **标注问题**（来源等级被冒充） | 数字无身份证 |
| ③ 假性裕量 / 假性完成（simulation_coverage_audit） | R1 等"纸面当落地" | **标注问题**（完成度被夸大） | 数字无身份证 |
| **④ PF-8 估测当实测（本复盘）** | **d=30 [L0] 目测当几何基线 LOCKED** | **标注问题 +「冲突处置」问题（新维度）** | 数字无身份证 **+ L1↔LOCKED 冲突无强制重审机制** |

**同根**：四者全部源于"**数字没有来源身份证**"——读者分不清一个数是测的、算的、估的还是猜的，于是低可信度的数被当高可信度用。这是 POLICY-PROV-001 的立制初心。

**PF-8 多出的新维度（必须单独记住）**：前三个都是"**一个数字本身**有问题"（标错/夸大/误信），靠"标好来源等级 + 独立复核"即可治。**PF-8 是"两个数字打架时，制度该怎么办"**——这是个**冲突处置**问题，光标注来源解决不了：哪怕 d=30 标了 [L0]、d=55 标了 [L1]，**若没有"L1 赢、且强制重审被它推翻的 LOCKED"这条规则，"并存"依然会发生**。所以 v1.2 必须额外加铁律四（冲突处置）和 C6（冲突自查），而不只是加一个 L0 标签。**标注解决"看得清"，重审解决"打架时谁说了算"——两者缺一不可。**

## 5. 正面教训（守门机制成熟的体现）

PF-8 是一次"挖出旧坑"的事件，但**处理 PF-8 的过程本身展示了守门机制已经成熟**，值得与 DEC-S3-PF4-01 正面案例并列沉淀为项目文化：

1. **PM 判据校准防新假性完成（栅瓣 3.1k vs 6.2k）**：PF-8 处理 d=55 栅瓣代价时，PM/Critic **没有把半波长判据 c/(2d)≈3118Hz 和严格判据 c/d≈6236Hz 搞混**。若错用 3.1k 当真问题带起点，会把 5–6kHz 安全带误判为栅瓣区，触发一轮不必要的规格大降级（新假性问题）。正确锁定"broadside 真问题带 = 6.2–8kHz、500Hz–5kHz 安全"[L1 判据]，**用对的判据防住了一个本来会自造的坑**——这是"不制造新假性完成"的主动质量行为。

2. **PM 主动识别工装/PCB 边界、不滥用 ESCALATE**：PF-8 衍生评估（P1-B）中，PM 准确区分"d 变化影响的是阵列**承载结构/工装**（结构域），而非自研**主控 PCB**（电气域，与 d 解耦）"，据此判定 **PCB 节奏不受影响、不触发 ESCALATE**——既没漏报（工装确需重做，已纳入时间表），也没为了"显得谨慎"而滥发 ESCALATE 惊动 CTO。**ESCALATE 的边界拿捏准确**，是守门机制成熟（而非过度防御）的标志。

3. **与 PF-4 同一文化线**：DEC-S3-PF4-01 是"Critic+PM 把三个误导数字拦在源头、零污染 LOCKED"的正面案例；PF-8 是"审计主动挖出旧坑、CTO 终局裁定、制度即时补丁"。两者共同说明：**项目已从"被动发现缺陷"进化到"主动审计 + 制度自愈"**。PF-8 暴露的是 Sprint 2 的历史债，而非当前流程的失效——当前流程恰恰是把这笔债识别、定性、清偿、并立法防复发的那套机制。

## 8. 制度实战检验附录（POLICY 从"立"到"用"的真实证据，2026-05-29）

> CTO 钦定收录。前面 §1-7 记"制度为何立、怎么补"；本附录记"制度立完之后，在真实任务里**是否真的生效**"——两个案例分别证明 **teammate 已内化 POLICY**（防患于派单阶段）与 **制度能自我检验、自我修复**（铁律五反噬到自己头上也照修）。

### 案例 A — C7 首次主动评审实战：teammate 在派单阶段自行内化 POLICY（防 PF-9 复发）
- **场景**：PF-9 闭环后立刻的"Sprint 3 仿真全景 + 竞品对比可行性"任务（`sprint3/audit/PF9_post_simulation_panorama.md`），是 C7/铁律五立法后的**首次主动评审**（此前 C7 都是清扫后复核）。
- **证据**：① 在**派单 prompt 阶段**就把"禁 SPL→BW 反推 / 禁撤回值 / 仅同口径"三铁规钉死——制度在任务设计层就生效，不等产出后再拦；② **acoustic-simulation teammate 自行拦下 3 个 PF-9 诱惑点**（竞品 §7 完整 SPL 极易插值反推 BW；竞品 106-111dB 易与我方 117dB[L4] 比绝对值；竞品 180° SPL 易凑"前后比"）——**未等 Critic，teammate 自己判 ❌ 并说明理由**；③ Critic C1/C2/C7 三门禁复核**全 PASS、零 BLOCKER、零新 PF-9 隐患**（`critic_panorama_gate.md`）。
- **意义**：制度成熟的最高信号不是"Critic 拦住了错"，而是 **teammate 在产出前就不犯错**——POLICY 从"外部门禁"内化为"teammate 的默认行为"。这是 DEC-S3-PF4-01（守门拦截）→ PF-8/9（主动审计+立法）之后的第三阶段：**预防内化**。
- **关联**：critic memory LESSON-011。

### 案例 B — 铁律五反噬自身：E-NEW-3 处置状态"未传播"被自身规则抓出并修复
- **场景**：E-NEW-3 全库清扫（PF-9 残留）执行完毕、C7 复核残留 0 之后，`simulation_coverage_audit.md` §2 PF-9 条目与页脚却**仍写"本次未执行清扫"**——清扫做了，但"已完成"这一**状态变更未传播**回 audit。
- **证据**：后续 panorama 任务的 Critic 读到该 stale 页脚，提"E-NEW-3 未结"；PM 核实发现是 audit 状态未同步（清扫实际已完成），随即更新 audit §2/页脚标"E-NEW-3 已执行、C7 PASS 残留 0"。
- **意义（讽刺而珍贵）**：这正是**铁律五（撤回/处置必须全库传播）反噬到制度自己头上**——"E-NEW-3 完成"是一个状态变更，同样需要全库传播，否则 stale 状态会误导下游（连 Critic 都被误导一次）。制度抓出了自己执行中的同型疏漏并修复，**证明铁律五不是只约束别人的口号，对制度自身的执行同样有效**。
- **教训固化**：状态变更（不只数值，含"已完成/已撤回/已作废"等**状态位**）也须按铁律五全库传播；"工单完成"的收尾定义应含"回填所有引用该工单状态的文档"。

### 案例 C — C7 第五次主动评审"首次抓到 BLOCKER"：证明门禁是实的、非橡皮图章（CTO 钦定记录，对照案例 A）
- **场景**：标准 12 点合规后的 R10 调研（A1 静态加权 / A2 FIB / A3 决策）。前四次 C7 主动评审（panorama / 竞品对比 / MATLAB 三轨 / 标准12点）**连续零 BLOCKER**——存在被质疑"门禁走过场/橡皮图章"的风险。
- **证据**：第五次评审（A1/A2/A3）中，Critic 在 `A2_fib_feasibility.md` line 91 **抓到一处 PF-9 复发**——FIB 报告把**已撤回的竞品插值 19.1°（F-AC-01）冠以红线词"实测"**作高频参照、且 point-of-use 无内联撤回警示（异地免责 line 165 不抵）。判 **2 BLOCKER（C2 措辞 + C7 撤回传播）**，A2 整体 FAILED → PM 修复（删"实测" + 内联"已撤回·非实测·不作对比锚点"警示）→ delta 复核转 PASS。
- **意义（与案例 A 对照）**：案例 A 证"teammate 产出前就不犯错（内化）"；案例 C 证"**真犯错时门禁真拦得住**"。两者合起来才完整——**零 BLOCKER 不等于门禁失效（可能内化），但门禁必须保留"真能抓"的能力**。第五次的这一抓，把"前四次零 BLOCKER"从"可能走过场"坐实为"确有内化 + 门禁仍锐利"。**制度对所有产出者平等**：连 FIB 战略预研这种高价值产出，撤回值误用照样被拦。
- **CTO 元层级承认（钦定记录）**：本轮 CTO 自承两次"凭印象判断"被纠正——① 未察"项目本有子带分频架构"致漏看"子带标量加深零成本关 R10"（PM 跑出）；② 此前两次（14.9/19.2 + 本案）CTO 自身引用/判断被 Critic 抓。**制度对 CTO 也平等 = 制度文化最大价值**。
- **关联**：LESSON-011（零 BLOCKER 内化，案例 A）+ LESSON-012（三轨验证 + 本次首抓 BLOCKER）；印证 POLICY v1.3 铁律五 + Critic C6/C7 落地有效。
