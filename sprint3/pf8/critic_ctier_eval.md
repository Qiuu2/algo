## 2. C 档两机制的当下必要性 vs 时机

### 机制 A：对外引用状态位机制（每个对外数字带 provenance 状态位）
- **价值方向正确**：管的是"内部→外部"边界，正是 PF-9 暴露的真实痛点（撤回值经外部 AI 再传播）。
- **当下时机错位**：项目尚无正式对外发布渠道；目前"对外"仅指与外部 AI 的研讨场景，而该场景的防护已由 **B 档 C7 出口门禁 + 铁律五**覆盖（撤回后全库残留=0 才可宣称已传播）。在无对外发布管线时建"对外状态位"机制，是为不存在的接口预先造规程，**易僵化、易随 Sprint 4 实际对外形态变更而返工**。
- **正确时机 = Sprint 4 首次对外触发点**（首份客户版规格书 / 首次对外性能承诺 / 首次展会）——届时对外数字的形态、渠道、责任人都确定，状态位设计才有落点。

### 机制 B：全项目数字身份证复审
- **与已完成工作高度重叠**：B 档已完成 ① 竞品 BW 全库扫（E-NEW-3，C7 实战 PASS、残留 0）② PF-1~PF-9 数字图谱（`PF_atlas.md`，含 L1–L4 标级 + 处置 + 状态）。
- **边际价值不足**：全项目复审与 atlas/audit 覆盖面重叠；当前数字资产规模有限（pre-PCB，无大量对外交付物），全量复审投入产出比低。
- **真正需要全量复审的时机**：EZKIT 实测 / T-S 回报后，大批 L3/L4 占位值（SPL 117dB、延迟 12.53ms、算力裕量）升级为 L1 实测——**那时数字身份证发生实质变更，复审才有新增对象**。现在复审的是同一批尚未变更的数字。

## 3. 风险权衡与建议

| 选项 | 残留风险 | 成本/时机 |
|------|---------|----------|
| **停 B** | 外部 AI 类误引可能再发——但 B 档 C7+铁律五已使"撤回真正生效（残留 0）"，再发时同一机制即可拦截/清扫；无客户/合同 → 误引不波及外部信任（CTO 判据核心顾虑不存在）| 零额外成本，不过早造规程 |
| **升 C（全量）** | 几乎无新增风险下降（C7 已堵 PF-9 主漏洞）| 为不存在的对外接口预造机制 + 与 atlas/audit 重叠的复审 → **过早、空转、易返工** |
| **部分升** | — | 不必要：两机制的有效落点都在 Sprint 4 对外触发点之后 |

**明确建议：停在 B 档。** 理由：
1. CTO 升 C 的**字面触发前提（对外有客户/PRD/合同）当前全部不成立**；
2. B 档 C7 出口门禁 + 铁律五已实战 PASS（E-NEW-3 残留 0），PF-9 主漏洞已堵，外部 AI 误引再发时同机制可处置；
3. C 档两机制（对外状态位 / 全量复审）的**有效落点均在 Sprint 4 首次对外触发点之后**，现在建只会僵化与返工，违反"不偏向多做"的客观原则。

---

## 4. 停 B 的触发条件（何时重启 C 评估）

满足**任一**即重启 C 档评估，由 PM 提交 CTO：
1. **首个外部客户接触 / 签约意向**（出现客户版规格沟通）；
2. **首份对外性能承诺生效**（DEC-S2-012 等"待市场对齐"规格转为正式对外 / 出现对客户版 PRD）；
3. **首次对外发布**（展会 / 公开样机演示 / 对外文案含具体性能数字）；
4. **EZKIT/T-S 实测回报**触发大批 L3/L4 → L1 升级（届时机制 B 全量复审才有新增对象，可一并评估）。

> 命中 1–3 任一 → 升 C 主体（对外状态位机制优先）；命中 4 → 评估机制 B（全量复审）。

---

## 5. 是否 ESCALATE

**否。** 本评估结论（停 B）与 CTO 判据完全一致（前提不成立则停 B），无跨域冲突、无安全/合规红线、无 CTO 级决策矛盾，属 Critic 权限内常规裁定。仅需 PM 知会 CTO 备案 + 登记触发条件 watch。
