## 草案 B — Critic C7 撤回传播门禁（拟并入 POLICY §7 C 表 + `agents/critic/skill.md` §11.2）

> **C7 撤回传播门禁**：任何数字被撤回时，Critic 必须强制**全库反向清扫**——grep 该数字（及其派生值/别名）的所有引用，对**每一处**确认"已加撤回警示标签"或"已删除"；清扫未完成前，**撤回声明不生效**。未通过 C7 的撤回 = **BLOCKER**。

**判级**：撤回声明已发出但全库残留 ≥1 处未加标/未删的引用 → **BLOCKER**（撤回视为无效，不得宣称"已撤回/已修正"）。

**Critic 检查动作**（每次涉及撤回的评审必做、并在报告显式给出结论）：
1. 取被撤回数字的字面值 + 所有派生值/反推值/别名（如 14.9° 及其插值来源、19.2° 自仿行）；
2. 全库 grep（含 `full_teardown*`、`sprint*_status.md`、decisions_log、对外交付物）；
3. 对**每处命中**判定：已加撤回警示标签 / 已删除 / **残留未处理（FAIL）**；
4. 任一处残留 → C7 FAIL → BLOCKER，撤回声明回退为"未生效"，打回补清扫。

**与铁律四 / C6 的关系**：
- **铁律四**（L1 实测与 LOCKED 决策冲突 → 强制重审，禁"并存"）管的是**冲突触发重审**；**C7** 管的是**重审/撤回的结论是否传播到全库**。铁律四负责"该改"，C7 负责"改干净、改全"——两者前后衔接，C7 是铁律四（及任何撤回）的**收尾闭环门禁**。
- **C6**（几何 LOCKED 必须 L1 / 冲突未重审 = BLOCKER）是**入口门禁**（防错误数字 LOCKED）；**C7** 是**出口门禁**（防已撤回数字残留传播）。C6 防进、C7 防赖着不走。

### ② POLICY 主体新增「撤回传播义务」条款（拟设为铁律五）
> **铁律五（v1.3 新增，缘起 PF-9）——撤回必须全库传播，三步缺一不可**：任何数字被撤回时，撤回动作 = ① **发出撤回声明**（注明被撤值、撤回原因、撤回令号如 F-AC-01）+ ② **全库反向扫描**该数字及其派生值/别名的所有引用 + ③ **逐处加撤回警示标签或删除**。三步未全部完成，撤回**不生效**，不得对外/对下游宣称"已撤回/已修正"。违者 BLOCKER（由 Critic C7 守门）。
