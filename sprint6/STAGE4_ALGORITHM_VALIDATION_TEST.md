# 定向音柱算法有效性 — 整机验证测试文档

> 目的：在**真实硬件链**（AD-EXKIT 跑 M2 → DAC → 功放 ×4(8路) → 16 元阵列）上，
> 系统性确认「M2 FIRA broadside 波束算法**真的产生了定向效果**」，并量化到可与仿真对账的程度。
> 这是阶段④ 软件收口后、阶段⑤ 自研 PCB 之前的**整机算法验证**——也是第一次用真功放+真阵列听到算法效果。
> 维护：PM lead。权威数据源：M2 board PASS（DEC-S6-M2-BOARD-PASS-01）+ Dolph 权重表 + 阵列因子仿真。

---

## 0. 先读：这份测试「证明什么 / 不证明什么」（L 分级与边界）

**证明（本测试的目标）**：
- 整条电-声链通（8 路都出声、无哑路/异响）；
- 8 路幅度锥化 = Dolph 设计（接线顺序/极性对）；
- **正前方有定向波束、侧面明显衰减**——即算法在真实声学上有效；
- 实测指向性与 [L2] 阵列因子仿真大致对账（量级一致）。

**不证明（诚实边界，别越界宣称）**：
- **不是** R3 消声室 L1 终验：本测试在普通房间，有反射/驻波，是 **[L1 现场实测 / 普通房间口径]**，半定量；绝对 SPL、精确波束宽度、≥90dB 对外承诺仍待 R3。
- **不是** bit-exact 数值正确性：逐子带 CRC（M2_Q_BOUNDARY_SURVEY §3）是另一道独立门，本测试不替代。
- 当前是 **broadside（正前固定波束）**，**不是**波束偏转，也**不是**多档扩散角（那些是后续立项）。

**关键数字来源标级**：
- Dolph 权重 / 各路相对电平 = **[L2 设计权重 / 确定性比值派生]**（g_dolph_w8_q15 桌面 Q15 量化值，header 自标 L2/dual-track；板上是「应用」该权重非「实测」出它；相对电平=纯 20log10 比值，不依赖来源等级）；
- 波束宽度 / 旁瓣 / 栅瓣 = **[L2 阵列因子仿真 + L1 几何]**（理想点源/自由场；与档案 AC-WP01 + 本项目 MATLAB 双轨一致）；
- M2 板上读数 = **[L1/EZKIT]**（DEC-S6-M2-BOARD-PASS-01）。

## 1. 被测系统与前置条件

| 项 | 配置 |
|---|---|
| DSP 板 | AD-EXKIT V2.1 + 21569-SOM，跑 **M2 build**（`M2_FIRA_INLOOP=1` + `FIRA_USE_REAL_ADI_FIR_HEADER`）|
| 算法 | M2 FIRA broadside v1：每路 = FIRA(输入 × Dolph 权重 w[c])，broadside 全同相，无偏转无聚焦 |
| 阵列 | 16 元线阵，d=55mm，L=825mm，竖直摆放，broadside 朝正前 |
| 功放 | 4 片 × 2 路 = 8 路（ACM3128A 等效）|
| 喇叭接法 | **每路功放 → 一个对称镜像串联对 {元件 c, 元件 15-c}**（已确认）|
| 信号链 | M2 第c路 → TX slot c → DAC 通道(c+1) → OUT[c] → 功放第c路 → 串联对{c,15-c} |

**前置确认**：
- M2 板测已 PASS（声音正常，DEC-S6-M2-BOARD-PASS-01）；
- 立体声音源接 AD-EXKIT 输入（IN1-2），最大音量（捕获 slot 吃 ADC ch2，必用立体声，见 runbook）。

**⚠ 安全（每次必读）**：第一次驱动真喇叭——**功放从音量最小或断电起步**，逐步上。若某帧 FIRA 异常会是近满幅噪声（板测 out_max_abs 未顶满幅已排除，但物理保险照做）。听到刺耳/破音立即降音量。

## 2. 测试项（按顺序，逐项 PASS/FAIL）

### 测试 0 — DSP 端在出正确波束（电域，不出声也能确认）
板上 Expressions 读，跟 1B PASS 一致即 PASS：

| 变量 | PASS 判据 |
|---|---|
| `g_m2_fira_inloop` | = 1 |
| `g_m2_setup_rc` / `g_m2_valid` / `g_m2_fg_beam_live` | 0 / 1 / **1** |
| `g_m2_out_nonzero` | 持续增长 |
| `g_m1_rx_block_count` ≈ `g_m2_poll_count` | 都增长且相等 |
| `g_m2_overrun_count` | **≈0（主判据：证波束每帧塞得下）** |
| `g_m2_beam_cyc_max` | < 1,333,333（帧周期；**板测基线≈830903≈62%墙钟，余量仅1.6×非「远低于」**；显著超~110万或 overrun 涨=异常。墙钟口径含等FIR忙等，勿与纯核~130µs[L2]比，R60纪律）|

→ 全绿 = DSP 在出正确的 8 路 Dolph 波束信号。**先过此项，再上功放。**

### 测试 1 — 接线顺序（Dolph 响度指纹）
放**稳定单音 2kHz**（或粉红噪声），功放小音量。测**每路/每对喇叭的相对响度**（手机分贝计贴近各对，或听感排序）。
应呈 Dolph 锥化形状 [L2 设计权重 / 比值派生]：

| 阵列位置（对）| 该路 c | 相对中心电平 [L2派生] |
|---|---|---|
| **最中心对** | c=7 | **0 dB（最响）** |
| 往外 | c=6 | −0.26 |
| | c=5 | −0.79 |
| | c=4 | −1.59 |
| | c=3 | −2.69 |
| | c=2 | −4.13 |
| 次外对 | c=1 | **−5.95 dB（最小）** |
| **最外对** | c=0 | **−1.24 dB（抬边，比次外明显响 +4.7dB）** |

**PASS 判据**（**只认这三个硬指纹**，手机分贝计 ±1-2dB 误差会吃掉中间档 c4-c6 的零点几 dB 差异，属正常）：
① **中心对(c7)最响**；② **次外对(c1)最小**；③ **最外对(c0)明显比次外对响**（抬边 +4.7dB）。中间档(c2-c6)单调递增是趋势参考、不作硬判据。
**FAIL**：中心不是最响 / 最外比次外小 → **顺序接反**，按 §1 表重排功放输入线序后重测。

### 测试 2 — 极性（broadside 正前最大）
放 **2kHz 单音**，麦/耳在阵列**正前方 1m、阵列中心高度**。

**PASS 判据**：正前方是**明显最大**（broadside 同相叠加）。
**FAIL**：正前方不是最大、或正前方有个**凹陷/抵消**（某处反相）→ **极性翻了**，查串联对接线 + 功放输入正负，逐对核对后重测。

### 测试 3 — 指向性 / 波束（★算法有效性的核心证据）
房间尽量空旷、远离墙（≥1.5m），阵列离地架高到中心高度。麦/耳固定距离 **1m**。
放 **2kHz 单音**，绕到这些角度各测一次 SPL（以正前 0° 为基准）：

| 角度 | 测什么 |
|---|---|
| 0°（正前）| 基准 SPL |
| ±30° | 相对 0° 落差 |
| ±60° | 相对 0° 落差 |
| ±90°（正侧）| 相对 0° 落差 |

**PASS 判据（算法有效的硬指标）**：
- **0° 最大**；
- **±90° 相对 0° 落差 ≥ 12 dB**（保守门；理论旁瓣 −20dB，留房间反射裕度）；
- 左右**大致对称**（±30°/±60°/±90° 左右两侧落差差 ≤ 6 dB）。

→ 满足 = **波束成立、定向有效、极性对称全对**。这是「算法在真实声学上有效」的实证。

### 测试 4 — 频率行为（印证物理，对账仿真）
分别放 **1k / 2k / 4k / 6k Hz** 单音，各测 0° 与 ±90°，并感受主瓣宽窄：

| 频率 | 仿真预期 -6dB 全角 [L2] | 现场预期感受 |
|---|---|---|
| 1kHz | 29.3° | 波束较宽 |
| 2kHz | 14.6° | 明显变窄 |
| 4kHz | 7.3° | 更窄 |
| 6kHz+ | 栅瓣（6.24kHz 起）| **侧面又冒声**（±50~90° 出现强瓣）|

**PASS 判据**：① 频率越高主瓣越窄（0° 越尖、侧面越低）；② **6kHz 以上侧面明显回升**（栅瓣，印证 d=55mm 上限）。
→ 这两条对上 = 实测物理与仿真一致，模型可信。

### 测试 5 —（可选，量化加分）SPL-vs-角度方向图
若有测量话筒：固定 1m，每 10° 测一圈（0°→±90°），画 SPL-vs-角度曲线，叠到本会话 [L2] 仿真方向图上。
**对账判据**：实测主瓣宽度与仿真同量级（普通房间允许 ±一档展宽），旁瓣被压低的趋势一致。

## 3. 综合判定 —「算法确认有效」的条件

全部满足即判 **算法整机有效 [L1 现场/普通房间口径]**：

| # | 条件 | 来自 |
|---|---|---|
| ✅ A | 测试 0 DSP 端全绿 | 测试 0 |
| ✅ B | 8 路都出声、无哑路/异响 | 测试 1 |
| ✅ C | Dolph 响度指纹对（c7中心最响 + c1次外最小 + c0抬边）| 测试 1 |
| ✅ D | 正前方最大、无反相凹陷 | 测试 2 |
| ✅ E | **±90° 落差 ≥ 12dB + 左右对称** | 测试 3 |
| ✅ F | 高频变窄 + 6kHz 栅瓣回升 | 测试 4 |

**E 是核心**（定向的硬证据），C/D 保证 E 不是接错的巧合。
**任一 FAIL** → 按对应测试的 FAIL 分支修（顺序/极性/接线），不下结论。

**诚实声明**：判「有效」= **方向性在真实硬件上成立且与仿真量级一致**；**不等于**消声室验收通过、不等于 bit-exact、不等于 ≥90dB 承诺达标——那些是 R3 / CRC 门 / SPL 解冻的独立事项。

## 5. 安全与红线
- 功放**最小音量/断电起步**，逐步上；刺耳/破音立即降。
- 不可逆动作（改接线）前断功放电。
- 任何异常**记录发回，不自行改算法/build**（M2 已 PASS，问题大概率在物理连线）。
- 本测试**仅整机验证**，无采购/PO 动作。
